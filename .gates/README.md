## 🚨 门禁检查内容

门禁系统会检查以下6个关键方面：

### 1. 测试真实性验证 🔴
- 检测是否包含模拟测试
- 检测是否用异步函数冒充真实进程
- 检测是否用进程内并发冒充多进程并发

### 2. 验证等级确认 🔴
- 检查是否明确标注验证等级
- 检查等级是否与实际测试相符
- 检查是否夸大验证程度

### 3. 局限性说明 🔴
- 检查是否明确说明测试局限性
- 检查是否说明未验证的场景
- 检查是否说明不确定性

### 4. 证据完整性 🔴
- 检查结论是否有证据支持
- 检查是否提供可验证的数据
- 检查是否提供可复现的步骤

### 5. 标题准确性 🔴
- 检查标题是否准确反映内容
- 检查是否有误导性词汇
- 检查关键术语是否准确

### 6. soul.md原则对齐 🔴
- 检查是否违反严苛验证第一原则
- 检查是否重演2026-03-22错误
- 检查是否诚实报告所有发现

## 📝 集成到工作流

### 在package.json中添加脚本

```json
{
  "scripts": {
    "gatekeeper": "node .gates/gatekeeper.js",
    "gatekeeper:check": "node .gates/gatekeeper.js",
    "pre-commit": "npm run gatekeeper"
  }
}
```

### 在Git hooks中使用

```bash
# .git/hooks/pre-commit
#!/bin/bash
echo "🔒 运行门禁检查..."
npm run gatekeeper -- ./docs/*.md
if [ $? -ne 0 ]; then
  echo "❌ 门禁检查未通过，禁止提交"
  exit 1
fi
```

### 在CI/CD中使用

```yaml
# .github/workflows/gatekeeper.yml
name: Gatekeeper Check
on: [pull_request]
jobs:
  gatekeeper:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run Gatekeeper
        run: |
          for file in docs/*.md; do
            node .gates/gatekeeper.js "$file"
          done
```
