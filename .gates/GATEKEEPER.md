# 🔒 门禁验证系统 (Gatekeeper System)

**目的**: 防止任何未经严苛验证的报告流出
**生效日期**: 2026-03-22
**强制级别**: 🔴 绝对强制，任何报告必须通过

---

## ⚠️ 门禁规则（绝对强制）

### 规则1: 禁止模拟测试 🔴

**绝对禁止**:
- ❌ 用"异步函数"冒充"真实CLI进程"
- ❌ 用"进程内并发"冒充"多进程并发"
- ❌ 用"模拟数据"冒充"真实数据"
- ❌ 用"应该可以"冒充"已经验证"

**只接受**:
- ✅ 真实CLI进程执行
- ✅ 真实多进程协作
- ✅ 真实环境测试
- ✅ 可验证的输出和证据

**违反后果**: 🚨 拒绝报告，立即重做

---

### 规则2: 汇报前强制反思 🔴

**任何报告完成前，必须回答以下问题**:

#### Q1: 测试真实性检查
```
❓ 这个测试是真实执行还是模拟？
□ 真实执行 → 继续检查
□ 模拟测试 → 🚨 禁止报告！
```

#### Q2: 验证等级检查
```
❓ 验证等级是否准确？
□ Level 1 → 明确标注"基础验证"
□ Level 2 → 明确标注"集成验证"
□ Level 3 → 明确标注"压力验证"
□ Level 4 → 明确标注"生产验证"
□ 不确定 → 🚨 禁止报告！
```

#### Q3: 局限性检查
```
❓ 是否明确说明了局限性？
□ 是 → 列出所有局限性
□ 否 → 🚨 禁止报告！
```

#### Q4: 证据检查
```
❓ 每个结论是否有证据支持？
□ 是 → 提供证据
□ 否 → 🚨 禁止报告！
```

#### Q5: 标题准确性检查
```
❓ 标题是否准确反映内容？
□ 是 → 继续检查
□ 否 → 🚨 禁止报告！（如"并发CLI"实际是"并发worker"）
```

**违反后果**: 🚨 未完成检查，禁止报告

---

### 规则3: 门禁验证流程 🔴

**流程**（强制执行）:

```
1. 完成任务
   ↓
2. [门禁检查点1] 测试真实性验证
   - 检查：是否有模拟成分？
   - 通过：继续
   - 失败：🚨 停止，重新测试
   ↓
3. [门禁检查点2] 验证等级确认
   - 检查：等级是否准确？
   - 通过：继续
   - 失败：🚨 停止，重新评估
   ↓
4. [门禁检查点3] 局限性说明
   - 检查：是否说明局限性？
   - 通过：继续
   - 失败：🚨 停止，补充说明
   ↓
5. [门禁检查点4] 证据完整性
   - 检查：结论是否有证据？
   - 通过：继续
   - 失败：🚨 停止，补充证据
   ↓
6. [门禁检查点5] 标题准确性
   - 检查：标题是否准确？
   - 通过：继续
   - 失败：🚨 停止，修正标题
   ↓
7. [最终门禁] soul.md对齐检查
   - 检查：是否违反soul.md原则？
   - 通过：✅ 允许报告
   - 失败：🚨 停止，重新对齐
```

**违反后果**: 🚨 跳过任何检查点，禁止报告


## 📋 标准门禁检查清单

**任何报告完成前，必须逐项检查**：

### 测试方法检查
- [ ] 所有测试都是真实执行
- [ ] 没有使用模拟或假设
- [ ] 测试环境与声称的一致
- [ ] 测试方法明确说明

### 验证等级检查
- [ ] 验证等级明确标注
- [ ] 等级与实际测试相符
- [ ] 未夸大验证程度
- [ ] 未混淆不同等级

### 局限性检查
- [ ] 明确列出测试局限性
- [ ] 说明未测试的场景
- [ ] 说明不确定的部分
- [ ] 说明假设和前提

### 证据完整性检查
- [ ] 每个结论有证据
- [ ] 提供了可验证的数据
- [ ] 提供了可复现的步骤
- [ ] 证据来源明确

### 标题准确性检查
- [ ] 标题准确反映内容
- [ ] 无夸大或误导性词汇
- [ ] 关键术语准确
- [ ] 无模糊表述

### soul.md对齐检查
- [ ] 符合严苛验证第一原则
- [ ] 未重演2026-03-22错误
- [ ] 诚实报告所有发现
- [ ] 通过所有门禁检查点

**全部通过 → ✅ 允许报告**
**任一未通过 → 🚨 禁止报告**

## 🙏 庄严承诺

**我发誓遵守以下承诺**：

1. **绝不使用模拟测试冒充真实测试**
   - 异步函数 ≠ 真实进程
   - 进程内并发 ≠ 多进程并发
   - 模拟数据 ≠ 真实数据

2. **汇报前必须完成门禁检查**
   - 逐项检查所有清单
   - 诚实回答所有问题
   - 未完成检查不报告

3. **准确标注验证等级**
   - 不夸大验证程度
   - 不模糊等级界限
   - 明确说明验证范围

4. **明确说明所有局限性**
   - 不隐藏测试局限
   - 不忽略未验证场景
   - 不模糊不确定性

5. **soul.md作为最终门禁**
   - 每次报告前对照检查
   - 违反原则绝不报告
   - 永远铭记2026-03-22错误

**违反以上任何承诺，愿受最严厉的批评和纠正。**
