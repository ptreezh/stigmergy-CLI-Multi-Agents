# Soul 自主决策功能规格

> Research output: autonomous decision-making system capabilities for Stigmergy Soul Agent
> Date: 2026-04-12
> Focus: DECI-01~06 requirements — decision framework, confidence thresholds, boundaries, audit, self-check, rollback
> Sources: .planning/PROJECT.md (Soul 自主决策最佳实践方案), SOUL.md, existing Soul codebase

---

## 背景与范围

Soul 当前只能执行预定义流程（evolve/reflect），遇到未定义场景时缺乏决策能力——要么盲目选择，要么卡死等待用户输入。

决策系统的核心价值：**Soul 在边界内自主行动，在边界外主动确认。**

---

## 表1：表-stakes（门槛功能）

**定义**：不实现则用户无法信任 Soul 的任何决策。任何一项缺失都直接导致用户损失。

| ID | 功能 | 复杂度 | 依赖 | 说明 |
|----|------|--------|------|------|
| TS-01 | **决策边界检测** | Medium | DECI-03 边界配置 | 每次决策前判断"在边界内还是边界外"。边界内自主行动，边界外暂停并询问 |
| TS-02 | **决策审计日志** | Low | 无 | 每次决策记录：输入、选项、选择、理由、时间戳。存储于 `.stigmergy/soul-state/decisions/` |
| TS-03 | **置信度阈值降级** | Medium | DECI-02 阈值配置 | 置信度 < 阈值时自动降级为询问用户，不盲目行动 |
| TS-04 | **越界主动确认** | Low | TS-01 | 检测到边界外时生成确认提示并等待答复，不卡死不越权 |
| TS-05 | **紧急回退机制** | Medium | TS-02 | 连续 N 次决策失败时自动回退到安全状态并通知用户 |

---

## 表2：差异化功能（让 Soul 决策可靠且透明）

**定义**：在表-stakes 之上，这些功能让 Soul 的决策能力显著优于普通 agent。

| ID | 功能 | 复杂度 | 依赖 | 说明 |
|----|------|--------|------|------|
| DF-01 | **决策自检（决策后验证）** | High | TS-02 | 决策执行后验证结果与预期一致性；不一致则触发反思或回退 |
| DF-02 | **多选项推理追踪** | Medium | TS-02 | 记录每个候选选项的分析过程，不只是最终选择理由 |
| DF-03 | **决策边界动态收紧** | High | TS-01, DF-01 | 某边界内连续失败 N 次后自动收窄边界，后续仅询问 |
| DF-04 | **决策历史模式学习** | High | TS-02 | 基于历史决策日志提取模式，优化置信度估算（v1手动配置边界，暂不启用自动学习） |
| DF-05 | **决策结果可解释输出** | Medium | TS-02 | 对用户可见的决策理由摘要（结构化输出），非技术人员可读 |
| DF-06 | **跨会话决策上下文** | Medium | Soul 双源记忆 | 将决策上下文存入记忆系统，供后续会话参考 |
| DF-07 | **自适应策略选择** | Medium | TS-05, TS-02 | 分析失败模式选择正确策略（替代现有 round-robin），减少无效尝试 |

## 表3：反功能（明确不构建）

**定义**：表面上诱人，但会损害 Soul 决策系统核心价值的功能。

| ID | 反功能 | 不构建理由 |
|----|--------|-----------|
| AF-01 | 多 Soul 投票共识 | 单 Soul 场景不需要；引入分布式共识复杂度远超收益 |
| AF-02 | 决策树自动学习/更新 | 需要大量历史数据且容易过拟合；当前版本手动配置边界更安全 |
| AF-03 | 决策结果 LLM 重述 | 过早优化；TS-02 结构化日志已足够，后续按需扩展 |
| AF-04 | 无条件自主行动（无边界检查） | 违背"边界内自主行动，边界外主动确认"核心价值 |
| AF-05 | 静默失败（不通知用户） | 违反 SOUL.md 诚信约束；所有失败必须可追溯 |
| AF-06 | 跨 Soul 实例共识决策 | DECI-01~06 作用域为单 Soul；多实例共识是未来版本 |
| AF-07 | 外部决策引擎依赖 | Soul 决策逻辑需自包含，不引入外部规则引擎 |

## 表4：功能依赖图

```
DECI-02/03 (配置) ──→ TS-01 (边界检测) ──┬──→ TS-04 (越界确认)
                                           ├──→ DF-03 (动态收紧) ──→ DF-01 (自检)
                                           └──→ DF-07 (自适应策略)

TS-02 (审计日志) ──┬── DF-02 (推理追踪)
                   ├── DF-05 (可解释输出)
                   └── DF-06 (跨会话上下文)

TS-03 (置信度降级) ──→ TS-05 (紧急回退)
```

## 表5：实施优先级

| 阶段 | 内容 | 交付物 |
|------|------|--------|
| **P0**（必做） | TS-01 + TS-02 + TS-04 + TS-05 | 决策边界检测 + 审计日志 + 越界确认 + 紧急回退 |
| **P1**（第一轮差异化） | TS-03 + DF-02 + DF-05 | 置信度降级 + 多选项推理追踪 + 可解释输出 |
| **P2**（第二轮差异化） | DF-01 + DF-03 + DF-06 + DF-07 | 决策自检 + 动态边界收紧 + 跨会话上下文 + 自适应策略 |

## 表6：与现有 Soul 进化系统的关系

| 决策功能 | 对接现有模块 | 接口说明 |
