# Wiki知识获取技能

## 触发条件
当用户请求涉及以下任一任务时，激活本技能：
- 搜索和获取专业资料
- 学习新领域知识
- 整合多源信息
- 构建知识图谱
- 文献调研和资料收集
- 信息验证和质量评估

## 核心获取流程

### 第一步：需求分析和搜索策略
1. 分析用户的知识需求
2. 确定搜索范围和关键词
3. 运行 `scripts/search_planner.py` 制定搜索策略
4. 选择合适的搜索源和工具
5. 设定质量标准和过滤条件

### 第二步：多源信息搜索
1. 使用web_search工具进行学术搜索
2. 搜索技术文档和官方资料
3. 查找专业论坛和社区资源
4. 收集实践案例和经验分享
5. 整合初步搜索结果

### 第三步：文献处理和提取
1. 运行 `scripts/document_processor.py` 处理文档
2. 提取PDF文档的文本和结构
3. 清理和格式化网页内容
4. 解析文档元数据和引用
5. 保存结构化数据

### 第四步：知识整合和验证
1. 运行 `scripts/knowledge_integrator.py` 整合知识
2. 交叉验证信息可靠性
3. 解决矛盾和冲突
4. 构建知识图谱和关联
5. 评估内容质量和时效性

## 具体实施指南

### 学术知识搜索
```
当用户说"搜索量子计算最新研究"时：
1. 运行: bash("python scripts/search_planner.py --topic '量子计算' --type 'academic'")
2. 执行: web_search("quantum computing research 2023 2024", source="academic")
3. 处理: bash("python scripts/document_processor.py --type 'academic'")
4. 整合: bash("python scripts/knowledge_integrator.py --validate 'true'")
5. 输出: 结构化的知识报告和引用列表
```

### 技术文档收集
```
当用户说"收集微服务架构文档"时：
1. 运行: bash("python scripts/search_planner.py --topic '微服务架构' --type 'technical'")
2. 执行: web_search("microservices architecture best practices", source="technical")
3. 处理: bash("python scripts/document_processor.py --type 'technical'")
4. 整合: bash("python scripts/knowledge_integrator.py --build-graph 'true'")
5. 输出: 技术文档集合和知识图谱
```

### 实践案例研究
```
当用户说"研究DevOps实践案例"时：
1. 运行: bash("python scripts/search_planner.py --topic 'DevOps' --type 'practical'")
2. 执行: web_search("DevOps case studies best practices", source="community")
3. 处理: bash("python scripts/document_processor.py --type 'practical'")
4. 整合: bash("python scripts/knowledge_integrator.py --extract-patterns 'true'")
5. 输出: 案例分析和最佳实践总结
```

## 支持工具和脚本

### 搜索规划器
- 文件：`scripts/search_planner.py`
- 功能：制定智能搜索策略
- 使用：`python scripts/search_planner.py --topic "主题" --type "类型"`

### 文档处理器
- 文件：`scripts/document_processor.py`
- 功能：处理各种格式的文档
- 使用：`python scripts/document_processor.py --source "源文件" --type "类型"`

### 知识整合器
- 文件：`scripts/knowledge_integrator.py`
- 功能：整合和验证知识
- 使用：`python scripts/knowledge_integrator.py --input "数据" --validate "true"`

### 搜索策略指南
- 文件：`references/search_strategies.md`
- 功能：提供各类搜索策略和方法
- 包含：学术搜索、技术搜索、社区搜索策略

## 搜索类型和策略

### 学术型搜索
**目标**：获取权威学术资料和研究进展
**搜索源**：
- 学术数据库（Google Scholar, arXiv, IEEE）
- 学术期刊和会议论文
- 研究机构技术报告
- 学者个人主页和CV

**搜索策略**：
- 使用专业术语和精确关键词
- 限定时间范围（近3-5年）
- 优先高影响因子期刊
- 关注引用次数和权威性

### 技术型搜索
**目标**：获取技术文档和实现指南
**搜索源**：
- 官方文档和API参考
- 技术博客和教程网站
- 开源项目文档
- 技术会议演讲资料

**搜索策略**：
- 搜索官方文档优先
- 查找实际代码示例
- 关注版本兼容性
- 收集社区最佳实践

### 实践型搜索
**目标**：获取实际案例和经验分享
**搜索源**：
- 专业论坛和问答平台
- 社区博客和分享
- 案例研究和分析报告
- 经验总结和教训

**搜索策略**：
- 关注实际应用场景
- 查找具体问题解决方案
- 收集失败案例和教训
- 重视可操作性建议

## 文档处理能力

### PDF文档处理
- **文本提取**：使用OCR和文本提取工具
- **结构解析**：识别标题、段落、表格
- **元数据提取**：获取作者、标题、摘要
- **引用提取**：解析参考文献和引用
- **图表处理**：提取图表信息和说明

### 网页内容处理
- **内容清洗**：去除广告和无关信息
- **结构化提取**：提取正文和导航结构
- **链接验证**：检查链接有效性
- **元数据收集**：获取发布时间和作者信息
- **多媒体处理**：处理图片和视频链接

### 文档格式转换
- **格式统一**：转换为标准Markdown格式
- **编码处理**：处理不同字符编码
- **图片处理**：提取和优化图片
- **表格解析**：转换表格为结构化数据
- **引用格式化**：统一引用格式

## 知识整合方法

### 信息验证策略
1. **多源验证**：交叉对比多个信息源
2. **权威性检查**：评估发布机构权威性
3. **时效性评估**：检查信息发布和更新时间
4. **逻辑一致性**：验证信息逻辑一致性
5. **专家观点对比**：对比领域专家观点

### 矛盾解决机制
1. **矛盾识别**：自动发现信息矛盾点
2. **原因分析**：分析矛盾产生的根本原因
3. **权重评估**：评估不同信息源的权重
4. **平衡表述**：提供平衡的多角度观点
5. **不确定性标注**：明确标注争议和不确定内容

### 知识图谱构建
1. **实体识别**：识别关键实体和概念
2. **关系抽取**：抽取实体间的关系
3. **层次组织**：构建层次化的知识结构
4. **关联分析**：分析概念间的关联强度
5. **可视化支持**：生成知识图谱可视化

## 质量评估标准

### 可靠性分级
- **高可靠性**：官方文档、同行评议论文
- **中等可靠性**：行业报告、权威技术博客
- **低可靠性**：论坛讨论、个人观点

### 内容质量指标
- **完整性**：信息覆盖的全面程度
- **准确性**：信息的正确性和精确度
- **时效性**：信息的新鲜程度和更新频率
- **相关性**：与主题的关联程度
- **可理解性**：表达的清晰度和易懂性

### 引用规范
```markdown
#### 学术论文
[作者姓名, "论文标题", *期刊名称*, 卷(期), 页码, 年份]

#### 技术文档
[组织名称, "文档标题", 版本号, 访问日期]

#### 网页内容
[网站名称, "页面标题", 访问日期]

#### 社区讨论
[用户名, "讨论标题", 平台名称, 访问日期]
```

## 智能化特性

- 根据主题类型自动调整搜索策略
- 基于搜索结果优化关键词
- 动态调整搜索源权重

### 自动分类
- 基于内容自动分类文档
- 提取关键词和标签
- 识别文档类型和领域

### 关联发现
- 自动发现概念关联
- 识别引用关系
- 构建知识网络
- 推荐相关资源

## 常见问题处理

### 搜索结果不足
1. 扩展搜索关键词范围
2. 调整搜索源组合
3. 放宽时间限制
4. 尝试不同的搜索策略

### 信息质量不佳
1. 提高搜索源质量要求
2. 增加验证步骤
3. 寻找更权威的信息源
4. 明确标注不确定性

### 文档处理失败
1. 尝试不同的处理工具
2. 手动检查文档格式
3. 分步骤处理复杂文档
4. 记录处理失败原因

## 技能边界和限制

- 搜索结果受限于可访问的搜索源
- 文档处理依赖格式兼容性
- 知识整合质量受源信息质量影响
- 需要网络连接和足够带宽
- 大量文档处理需要时间
