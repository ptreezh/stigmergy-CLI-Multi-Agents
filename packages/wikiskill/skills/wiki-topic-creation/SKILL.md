# Wiki主题创建技能

## 触发条件
当用户请求涉及以下任一任务时，激活本技能：
- 创建新的Wiki主题或词条
- 建立知识框架和结构
- 组织和规划内容层次
- 设计章节结构和导航
- 定义核心概念和术语
- 生成主题模板和范例

## 核心创建流程

### 第一步：主题分析和规划
1. 分析用户需求和主题范围
2. 确定目标受众和使用场景
3. 运行 `scripts/topic_analyzer.py` 进行深度分析
4. 识别核心概念和关键术语
5. 规划内容深度和广度

### 第二步：结构设计
1. 参考 `references/topic_template.md` 选择合适模板
2. 设计章节层次结构
3. 确定导航和链接关系
4. 规划内容组织方式
5. 制定模板和样式规范

### 第三步：内容生成
1. 使用web_search工具收集相关资料
2. 生成核心概念定义
3. 创建详细章节内容
4. 添加实例和案例研究
5. 整合参考资料和链接

### 第四步：质量优化
1. 运行 `scripts/quality_checker.py` 进行质量评估
2. 验证内容准确性
3. 检查结构完整性
4. 测试链接有效性
5. 确保格式一致性

## 支持工具和脚本

### 主题分析器
- 文件：`scripts/topic_analyzer.py`
- 功能：分析主题特征和需求
- 使用：`python scripts/topic_analyzer.py --topic "主题名称" --type "主题类型"`

### 内容生成器
- 文件：`scripts/content_generator.py`
- 功能：基于模板生成结构化内容
- 使用：`python scripts/content_generator.py --template "模板名称" --topic "主题名称"`

### 质量检查器
- 文件：`scripts/quality_checker.py`
- 功能：评估生成内容的质量
- 使用：`python scripts/quality_checker.py --content "内容文件"`

### 主题模板
- 文件：`references/topic_template.md`
- 功能：提供标准化的主题结构模板
- 包含：基础结构、扩展结构、格式规范

## 主题类型和模板

### 技术型主题
**适用场景**：编程语言、框架、工具、技术方法
**核心要素**：
- 技术原理和基础概念
- 实现方法和代码示例
- 性能分析和优化
- 工具链和生态系统
- 最佳实践和经验

### 学术型主题
**适用场景**：理论概念、研究方法、学术领域
**核心要素**：
- 理论基础和文献综述
- 研究方法和实验设计
- 数据分析和结果讨论
- 学术贡献和创新点
- 未来研究方向

### 产品型主题
**适用场景**：产品介绍、功能说明、使用指南
**核心要素**：
- 产品概述和定位
- 功能特性详细介绍
- 使用方法和操作指南
- 应用场景和案例
- 竞争分析和市场定位

### 实践型主题
**适用场景**：操作指南、最佳实践、经验总结
**核心要素**：
- 操作流程和步骤
- 技巧和注意事项
- 常见问题和解决方案
- 经验分享和案例分析
- 工具推荐和资源整理

## 内容质量标准

### 准确性标准
- 所有技术信息必须准确无误
- 概念定义要科学严谨
- 数据和统计要有可靠来源
- 引用文献要权威可信

### 完整性标准
- 覆盖主题的核心方面
- 提供必要的背景信息
- 包含实际应用案例
- 提供进一步学习资源

### 清晰性标准
- 使用简洁明了的语言
- 逻辑层次清晰有序
- 专业术语要有解释
- 复杂概念要有示例

### 结构性标准
- 章节安排合理有序
- 标题层次清晰明确
- 导航链接准确有效
- 格式规范统一一致

## 智能化特性

### 自动化内容生成
- 根据主题类型自动选择模板
- 智能填充章节内容
- 自动生成导航链接
- 智能推荐相关资源

### 适应性结构调整
- 根据主题复杂度调整结构
- 根据目标受众调整深度
- 根据应用场景调整重点
- 根据可用资料调整范围

### 质量自动检查
- 自动检查内容完整性
- 智能验证链接有效性
- 自动检测格式规范
- 提供改进建议和优化方案

## 常见问题处理

### 主题范围过大
1. 分解为多个子主题
2. 建立主题层次结构
3. 创建主题关联网络
4. 提供分步创建方案

### 资料信息不足
1. 扩大搜索范围和关键词
2. 寻找权威和可靠来源
3. 明确标注信息缺失
4. 建议进一步研究方向

### 内容组织困难
1. 使用标准模板结构
2. 参考类似主题组织
3. 咨询领域专家意见
4. 收集用户反馈建议

## 技能边界和限制

- 生成内容质量依赖可用资料
- 需要用户进行最终审核确认
- 复杂主题可能需要分步创建
- 特定领域知识可能需要专家补充
- 时效性信息需要定期更新

## 扩展和自定义

### 添加新模板类型
1. 在 `references/` 目录创建新模板
2. 定义主题类型和特征
3. 更新模板选择算法
4. 测试模板效果

### 自定义质量标准
1. 修改 `scripts/quality_checker.py`
2. 调整评估指标和权重
3. 添加领域特定检查项
4. 更新质量报告格式

### 集成外部资源
1. 添加新的数据源接口
2. 集成专业知识库
3. 连接实时信息源
4. 更新资源整合逻辑
