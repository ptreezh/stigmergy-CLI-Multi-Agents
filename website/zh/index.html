核心特性

Stigmergy协作机制

基于共享项目背景的间接协同，无需中央协调器

内部自然语言交互

在任何CLI工具内部直接使用自然语言调用其他工具

原子性协作安全

防止多智能体同时认领相同任务的并发安全机制

任务状态同步

通过项目背景文件实现跨工具任务状态管理

无缝集成

不修改原始CLI工具，通过PATH优先级增强功能

去中心化架构

无单点故障，支持分布式协作和容错机制

工作原理

Stigmergy协作机制

  1. 项目背景初始化

    首次在项目目录使用协作功能时自动创建PROJECT_SPEC.json

  2. 协作意图识别

    Claude识别"让gemini帮我翻译"中的协作意图

  3. 任务创建到背景

    在PROJECT_SPEC.json中创建协作任务记录

  4. 智能路由执行

    自动路由到gemini执行翻译任务

  5. 状态更新同步

    任务状态更新到共享背景文件，供后续协作使用

安装指南

一键安装
# 克隆项目
$ git clone https://github.com/your-username/smart-cli-router.git
$ cd smart-cli-router
$ python INTERNAL_CLI_ENHANCER.py
# 配置PATH优先级
# Windows: 将 %USERPROFILE%\bin 添加到PATH开头
# Unix/Linux: 将 ~/bin 添加到PATH开头

安装完成后，所有支持的CLI工具都将获得协作能力：

  • Claude、Gemini、Kimi、Qwen等
  • Ollama、CodeBuddy、QoderCLI等
  • 自动发现系统中已安装的工具

使用示例

简单协作
$ claude "让gemini帮我翻译这份文档"
🔄 智能路由到: gemini

多工具协作
$ qwen "让claude分析需求，让gemini翻译文档"

代码生成协作

📋 PROJECT_SPEC.json 已更新
复杂项目协作

🔄 创建多阶段协作任务
📋 任务状态在PROJECT_SPEC.json中跟踪
