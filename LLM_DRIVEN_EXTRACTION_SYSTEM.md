# 经验层级 3：详细说明
### 问题分析
原代码使用了多个await语句，导致异步任务串行执行。
如果有3个各需1秒的任务，总耗时为3秒而不是1秒。

### 实施步骤
1. 识别哪些任务是独立的（无依赖关系）
2. 将这些任务包装为Promise
3. 使用Promise.all([task1, task2, task3])并行执行
4. 使用await等待所有任务完成

### 代码示例
\`\`\`javascript
// 串行执行（慢）
const result1 = await task1(); // 1秒
const result2 = await task2(); // 1秒
const result3 = await task3(); // 1秒
// 总计: 3秒

// 并行执行（快）
const [result1, result2, result3] = await Promise.all([
  task1(),
  task2(),
  task3()
]);
// 总计: 1秒
\`\`\`

### 验证方法
- 测量执行时间
- 验证任务独立性
- 确认结果正确性

# 异步性能优化技能

## 用途
当需要优化包含多个异步操作的代码性能时使用此技能。

## 能力
- 识别可并行化的异步操作
- 应用Promise.all()进行优化
- 处理错误和边界情况
- 提供性能测量方法

## 使用方法

### 识别优化机会
查看代码中是否有以下模式：
\`\`\`javascript
const result1 = await asyncOperation1();
const result2 = await asyncOperation2();  // 独立但串行
const result3 = await asyncOperation3();
\`\`\`

### 应用优化
\`\`\`javascript
const [result1, result2, result3] = await Promise.all([
  asyncOperation1(),
  asyncOperation2(),
  asyncOperation3()
]);
\`\`\`

## 示例

### 优化前
\`\`\`javascript
// 耗时: 3秒
const user = await getUser(id);
const posts = await getPosts(userId);
const comments = await getComments(postIds);
\`\`\`

### 优化后
\`\`\`javascript
// 耗时: 1秒 (最快任务的时间)
const [user, posts, comments] = await Promise.all([
  getUser(id),
  getPosts(userId),
  getComments(postIds)
]);
\`\`\`

## 核心知识

### 关键概念
- **Promise.all()**: 并行执行多个Promise
- **独立性**: 任务间无依赖关系
- **错误处理**: 使用try-catch或.allSettled()

### 最佳实践
1. 确保任务真正独立
2. 考虑使用.allSettled()处理部分失败
3. 注意内存占用（并行会同时创建多个操作）
4. 添加适当的错误处理

### 常见模式

#### 模式1: 数据获取并行化
\`\`\`javascript
// 串行（慢）
const user = await fetchUser();
const posts = await fetchPosts();
const comments = await fetchComments();

// 并行（快）
const [user, posts, comments] = await Promise.all([
  fetchUser(),
  fetchPosts(),
  fetchComments()
]);
\`\`\`

#### 模式2: 批量操作
\`\`\`javascript
// 串行处理
for (const item of items) {
  await processItem(item);
}

// 并行处理
await Promise.all(items.map(item => processItem(item)));
\`\`\`
