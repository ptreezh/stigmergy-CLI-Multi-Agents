# Wiki协同编辑技能

## 触发条件
当用户请求涉及以下任一任务时，激活本技能：
- 创建或编辑Wiki词条
- 知识体系构建
- 多源信息整合
- Wiki主题管理
- 协作编辑

## 核心工作流程

### 第一步：任务理解
1. 分析用户需求和任务目标
2. 识别Wiki主题类型
3. 确定创建或编辑策略
4. 评估任务复杂度

### 第二步：内容生成
1. 使用web_search工具获取相关资料
2. 整合多源信息
3. 生成结构化Wiki内容
4. 创建单网页Wiki文件

### 第三步：质量优化
1. 内容准确性验证
2. 结构优化
3. 格式美化
4. 生成最终Wiki页面

## 具体实施指南

### 创建新Wiki主题
```
当用户说"创建机器学习Wiki"时：
1. 分析任务：识别主题为"机器学习"
2. 搜索资料：使用web_search搜索机器学习相关内容
3. 生成内容：创建包含以下结构的Wiki页面：
   - 标题和简介
   - 核心概念
   - 主要内容章节
   - 参考资料
4. 创建HTML文件：生成完整的单网页Wiki
5. 返回结果：提供Wiki文件路径和预览
```

### 编辑现有Wiki
```
当用户说"编辑Python Wiki添加新章节"时：
1. 定位文件：查找现有Python Wiki文件
2. 分析内容：理解现有结构
3. 搜索新内容：获取要添加的章节资料
4. 整合内容：将新内容整合到现有Wiki
5. 更新文件：保存更新后的Wiki页面
```

### 知识整合
```
当用户说"整合AI知识体系"时：
1. 规划结构：设计AI知识体系框架
2. 分块搜索：分别搜索各个子领域
3. 内容整合：将各部分内容整合为完整Wiki
4. 关联建立：创建内部链接和引用
5. 生成Wiki：创建完整的知识体系Wiki
```

## Wiki页面结构

### 标准结构
```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>主题名称</title>
  <style>CSS样式</style>
</head>
<body>
  <header>标题和元信息</header>
  <nav>导航菜单</nav>
  <main>主要内容</main>
  <footer>页脚信息</footer>
  <script>交互功能</script>
</body>
</html>
```

### 内容组织
1. **标题区域**
   - 主标题
   - 创建时间
   - 作者信息
   - 最后修改时间

2. **导航区域**
   - 目录
   - 快速链接
   - 搜索功能

3. **主要内容**
   - 简介
   - 核心章节
   - 详细内容
   - 示例和案例

4. **页脚信息**
   - 参考资料
   - 相关链接
   - 标签和分类
