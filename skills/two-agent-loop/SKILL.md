# Two-Agent Loop - 双 Agent 循环

## ⚠️ 核心基础机制

**这是所有进化的基础机制。自主进化、协同进化、竞争进化都依赖此模式。**

```
┌─────────────────────────────────────────────────────────┐
│           双 Agent 循环 (Two-Agent Loop)                │
├─────────────────────────────────────────────────────────┤
│  主 Agent (当前 CLI)                                    │
│    │                                                    │
│    ├──→ 1. 派发 Subagent (执行具体任务)                │
│    │         │                                          │
│    │         └──→ 执行任务 → 返回结果                   │
│    │                                                    │
│    ├──→ 2. Code Review (质量审查)                      │
│    │         │                                          │
│    │         └──→ 反馈 → 改进                          │
│    │                                                    │
│    ├──→ 3. 迭代修复 (如有问题)                         │
│    │                                                    │
│    └──→ 4. 循环直到完成                                │
└─────────────────────────────────────────────────────────┘
```

## 执行流程

### 步骤 1: 任务分解

```
分析输入任务，分解为可执行的子任务：
- 每个子任务应该是原子性的
- 明确子任务的输入输出
- 评估复杂度
```

### 步骤 2: 派发 Subagent

```
使用 Task 工具派发 subagent：
- description: 子任务描述
- prompt: 完整的子任务指令
- 确保 subagent 有足够上下文
```

### 步骤 3: 执行与反馈

```
Subagent 返回结果后：
- 立即进行 Code Review
- 如有问题，派发修复 subagent
- 无问题则继续下一子任务
```

### 步骤 4: 迭代循环

```
重复步骤 2-3 直到所有子任务完成：
- 每个子任务都有质量关卡
- 问题不过夜（立即修复）
- 最终进行整体审查
```

## 使用场景

所有复杂任务都应该使用双 Agent 循环：

| 场景     | 子任务示例                                  |
| -------- | ------------------------------------------- |
| 自主进化 | 分析模式 → 提取知识 → 创建技能 → 测试验证   |
| 协同进化 | 收集知识 → 整合知识 → 融合提升              |
| 竞争进化 | 方案A执行 → 方案B执行 → 对比分析 → 选择最优 |
| 代码实现 | 实现功能 → 编写测试 → 代码审查 → 修复问题   |
| 系统设计 | 需求分析 → 架构设计 → 详细设计 → 评审       |

## 重要原则

1. **每个子任务后必须审查** - 不能跳过 Code Review
2. **问题立即修复** - Critical 问题不过夜
3. **不并行派发** - 避免冲突，按顺序执行
4. **使用 TDD** - Subagent 应遵循测试驱动开发
