System Architecture Explanation

1. User Interface Layer (Blue)

  • Stigmergy CLI - Main entry point for all user interactions
  • CLI Router - Processes user commands and routes them appropriately
  • Setup & Deployment - Handles system initialization and hook deployment

2. Core Coordination Layer (Green)

  • Node.js Coordination Layer - Central controller managing all cross-CLI communications
  • Hook Deployment Manager - Generates and deploys hooks to external CLI tools
  • CLI Communication - Executes cross-CLI tasks and manages process lifecycle
  • Adapter Manager - Manages tool adapters and provides unified interfaces
  • CLI Integration Manager - Handles integration specifics for each CLI tool
  • CLI Help Analyzer - Analyzes CLI help docs to understand parameter patterns
  • CLI Parameter Handler - Generates appropriate arguments for each CLI tool

3. External AI CLI Tools (Red)

  • 8 supported AI CLI tools that work collaboratively through the Stigmergy system
  • Each tool operates independently but can be orchestrated through hooks

4. Hook/Adapter Components (Orange)

  • Each external CLI tool has a corresponding hook deployed to it
  • Hooks detect cross-CLI requests and communicate with the coordination layer
  • Hooks are generated by the Hook Deployment Manager and stored locally

5. Data/Configuration (Purple)

  • Configuration Files - System-wide settings in ~/.stigmergy/
  • Hook Storage - Deployed hooks stored in ~/.stigmergy/hooks/
  • CLI Patterns Cache - Cached analysis of CLI tool parameter patterns
  • Documentation - Auto-generated documentation for each CLI tool

Cross-CLI Communication Flow

  1. User interacts with any supported AI CLI tool
  2. Hook in that CLI tool detects cross-CLI request pattern (e.g., "use qwen to...")
  3. Hook directly calls the CLI Communication module via relative path
  4. CLI Communication module generates appropriate arguments using CLI Parameter Handler
  5. If needed, CLI Help Analyzer dynamically analyzes the target CLI's help docs
  6. Target CLI tool is executed with proper parameters
  7. Results are returned through the hook to the originating CLI tool

Key Features

  • No Background Services - Direct module calls eliminate need for running services
  • Dynamic Parameter Learning - System learns CLI parameter patterns by analyzing help docs
  • Fault Tolerance - Multiple fallback mechanisms ensure system reliability
  • Distributed Hooks - Hooks deployed directly to each CLI tool for seamless integration
  • Unified Interface - Single coordination layer manages all cross-CLI communications
