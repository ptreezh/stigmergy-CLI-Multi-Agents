# 项目存档目录说明

本目录包含项目中不再使用的冗余文件和历史版本，按类别整理如下：

## 📁 目录结构

### `obsolete_docs/` - 废弃文档
- 各种版本的安装指南和部署文档
- TDD报告和技术验证报告
- 旧版README和项目说明文档
- 发布相关的临时文档

### `obsolete_scripts/` - 废弃脚本
- 旧版本的安装脚本 (install*.bat, install*.sh)
- 部署和验证脚本
- 演示和测试脚本

### `obsolete_configs/` - 废弃配置
- 旧版本配置文件
- 临时配置文件

### `demo_scripts/` - 演示脚本
- 功能演示脚本
- 工作示例脚本

### `reports/` - 报告文件
- 部署验证报告
- 测试报告

### `obsolete_files/` - 历史文件
- 完整的历史版本文件
- 旧版本的源代码和测试文件
- 网站相关文件

## 🗑️ 清理原则

以下文件已被移动到此存档：
1. **重复功能** - 多个版本实现相同功能的文件
2. **过时文档** - 已被新版本替代的文档
3. **临时文件** - 开发过程中的临时脚本和配置
4. **演示代码** - 仅用于演示的代码
5. **测试报告** - 已完成的测试报告

## 📋 保留的核心文件

根目录下仅保留以下核心文件：
- `README.md` - 项目主要说明文档
- `CLAUDE.md` - Claude Code 项目指导文档
- `INSTALL.md` - 安装说明
- `INTEGRATION_GUIDE.md` - 集成指南
- `PROJECT_SPEC.json` - 项目规范
- `setup.py` - Python 包配置
- `package.json` - Node.js 配置
- 核心源代码目录 (`src/`)
- 测试目录 (`tests/`)
- 文档目录 (`docs/`)

## ⚠️ 注意事项

- 存档文件不会被版本控制系统忽略
- 如需恢复某文件，请从此目录查找
- 建议定期清理此目录中确认不再需要的文件
- 重要配置和技术文档已移动到 `docs/` 目录
