快速安装

1

使用全局安装脚本

通过全局安装脚本检测并增强系统中所有可用的CLI工具

# 克隆项目
git clone https://github.com/ptreezh/stigmergy-CLI-Multi-Agents.git
cd stigmergy-CLI-Multi-Agents

# 运行全局部署
python deploy.py --global-setup

或使用 iFlow CLI 辅助安装（如果已安装）

iflow --install smart-cli-router
# 系统自动检测并为所有可用CLI工具添加协作能力
2

系统环境检测

安装脚本会自动扫描系统中已安装的AI CLI工具

  • ✅ Claude系列工具
  • ✅ Gemini系列工具
  • ✅ Kimi/Moonshot工具
  • ✅ Qwen/通义千问工具
  • ✅ Ollama本地模型
  • ✅ CodeBuddy、QoderCLI等代码工具
  • ✅ iFlow智能工作流助手
  • ✅ GitHub Copilot等其他工具
3

CLI智能体增强

为检测到的每个CLI工具添加Stigmergy协作能力

# 原始CLI工具获得协作能力
claude --enhanced # 自动获得协作功能
gemini --enhanced # 自动获得协作功能
qwen --enhanced # 自动获得协作功能
4

PATH优先级配置

配置增强版本的CLI工具在系统PATH中优先于原始版本

# Windows系统
# 在系统环境变量中将 %USERPROFILE%\smart_cli\bin 放在原始工具路径之前

# Unix/Linux/MacOS系统
export PATH="$HOME/smart_cli/bin:$PATH" # 添加到.bashrc或.zshrc中

Stigmergy（间接协同）机制详解

通过项目背景文件实现真正的去中心化智能体协作

什么是Stigmergy？

Stigmergy是一种去中心化的协作机制，智能体不直接通信，而是通过修改共享环境来间接影响彼此的行为。 在我们的系统中，项目目录背景（特别是PROJECT_SPEC.json文件）充当这种共享环境，使得多智能体能够基于环境线索自主协作。

Stigmergy协作流程


用户
→

Claude
→

Gemini
→

CodeBuddy

工作原理

  • • 环境感知 - 智能体读取项目背景文件中的待处理任务
  • • 自主决策 - 智能体基于任务类型和自身能力自主决定是否处理
  • • 环境修改 - 智能体更新任务状态、创建新任务或记录协作历史
  • • 间接通信 - 其他智能体通过读取环境变化了解协作状态
  • • 状态持久化 - 所有协作状态通过背景文件持久化存储

协作背景文件

系统在每个项目目录中自动创建协作背景文件:

PROJECT_SPEC.json # 任务状态、协作历史、当前状态
PROJECT_CONSTITUTION.md # 项目协作规则和宪法
COLLABORATION_LOG.md # 详细协作日志
TASKS.md # 人类可读的任务列表

使用示例

内部自然语言协作

在任何原始CLI工具内部直接使用自然语言指令：

# 在Claude内部发起协作
$ claude "让gemini帮我翻译这份README文档"
$ claude "请codebuddy优化这段Python代码"

# 在Gemini内部发起协作
$ gemini "用qwen写一段中文内容"
$ gemini "让claude审查这个方案"

# 在Qwen内部发起协作
$ qwen "让gemini翻译成英文"
$ qwen "请codebuddy帮我生成API文档"

多阶段协作

通过项目背景实现复杂的多阶段协作：

# 复杂协作指令
$ claude "让qwen写代码，让codebuddy审查，让gemini生成测试用例"

# 系统自动创建协作任务并分配给相应智能体
# 每个智能体基于PROJECT_SPEC.json自主执行任务

协作状态监控

通过背景文件监控协作状态：

$ cat PROJECT_SPEC.json | jq '.tasks' # 查看所有任务状态
$ cat TASKS.md # 查看人类可读任务列表
$ cat COLLABORATION_LOG.md # 查看协作历史日志
