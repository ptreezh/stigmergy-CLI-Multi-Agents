# Smart CLI Router 异构智能体协作系统文档

## 概述

Smart CLI Router 不仅仅是一个CLI工具路由器，它还是一个异构智能体协作系统。通过统一的项目规范文档和基于项目目录的间接协同机制，多个CLI工具（智能体）可以协作完成复杂任务。

## 核心概念

### 1. 统一项目规范文档
- 所有智能体通过 `PROJECT_SPEC.json` 文件进行协作
- 文档包含任务列表、状态信息、协作历史和决策记录
- 实现了真正的间接协同，无需复杂的通信协议

### 2. 智能体协作宪法
- `PROJECT_CONSTITUTION.md` 定义了协作的基本规则
- 规定了智能体的职责、协作机制和流程
- 可在不同项目间复制和扩展

### 3. 基于目录的协作
- 所有智能体共享同一项目目录
- 通过读写项目文件实现间接协同
- 保持协作过程的透明和可追溯

## 协作机制

### 任务委派
当一个智能体遇到无法解决或不确定的问题时：
1. 在项目规范文档中创建新的委派任务
2. 指定目标智能体类型和任务详情
3. 更新任务状态为"委派"

### 头脑风暴
多智能体讨论复杂问题时：
1. 创建讨论任务到规范文档
2. 各智能体贡献观点和建议
3. 汇总形成决策方案

### 分工协作
- 每个智能体专注自己的专业领域
- 通过规范文档了解其他智能体的工作状态
- 根据项目需要动态调整协作策略

## 使用方法

### 初始化协作项目
```bash
# 创建项目目录
mkdir my_collaboration_project
cd my_collaboration_project

# 复制项目宪法和规范文档
cp /path/to/smart-cli-router/PROJECT_CONSTITUTION.md ./
cp /path/to/smart-cli-router/PROJECT_SPEC.json ./
```

### 启动智能体协作
```bash
# 使用智能路由器执行协作任务
smart_ai.cmd "请帮我创建一个完整的Web应用，包含前端、后端和数据库设计"
```

### 监控协作过程
- 检查 `PROJECT_SPEC.json` 中的任务状态
- 查看协作历史记录
- 根据需要调整任务分配

## 智能体职责

### 任务识别
- 识别自己可以处理的任务
- 识别需要其他智能体处理的任务

### 状态更新
- 及时更新任务状态
- 记录处理结果和决策依据

### 协作沟通
- 通过规范文档与其他智能体协作
- 遇到困难时主动委派或请求协助

## 应用场景

### 1. 复杂项目开发
- 一个智能体负责架构设计
- 另一个智能体负责代码生成  
- 第三个智能体负责测试和优化

### 2. 多领域问题解决
- 不同专业领域的智能体协作解决复杂问题
- 知识互补，能力增强

### 3. 持续集成和部署
- 自动化构建、测试、部署流程
- 多智能体协同完成CI/CD任务

## 优势特点

### 简洁高效
- 无需复杂的通信协议和共享上下文系统
- 基于简单文件系统的协作机制
- 降低系统复杂度

### 可扩展性
- 任何CLI工具都可以作为智能体加入协作
- 无需修改现有工具
- 支持跨厂商、跨平台的AI工具集成

### 持久化协作
- 协作过程完全持久化存储
- 系统重启后可继续之前的协作任务
- 提供完整的协作历史记录

## 总结

Smart CLI Router 的异构智能体协作系统通过"统一项目规范文档 + 基于目录的间接协同"实现了真正的多智能体协作。这种架构突破了传统智能体协作系统的复杂性，提供了简洁、高效、可扩展的协作方案，为人工智能协作系统提供了一个创新的架构范式。
