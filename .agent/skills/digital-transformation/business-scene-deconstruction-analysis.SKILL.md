# 业务场景解构分析技能

## 何时使用
当需要全面分析业务场景，识别数字化转型机会，为制定数字化转型战略提供依据时使用此技能。

## 核心功能
1. **价值流解构**: 分析价值创造、传递、捕获的完整流程
2. **资源要素解构**: 识别有形、无形、人力、组织资源
3. **活动网络解构**: 梳理核心、支持、管理、连接活动
4. **连接关系解构**: 分析物理、信息、价值、认知连接
5. **生态环境解构**: 评估技术、市场、政策、社会环境
6. **创新机会识别**: 发现效率、体验、模式、生态缺口

## 执行步骤
1. 接收业务场景描述和分析需求
2. 从六大专家视角解构业务场景
3. 识别各维度的转型机会和痛点
4. 整合多维分析发现创新利基
5. 评估利基的战略价值和可行性
6. 生成业务场景解构分析报告

## 解构分析框架（DEEP-SCAN）

### 1. 价值流解构层 (Value Flow Decomposition Layer)
- **价值创造**: 从原材料到最终价值的完整转化路径
- **价值传递**: 价值在不同主体间的传递机制
- **价值捕获**: 各参与方的价值获取方式
- **价值分配**: 价值在生态中的分配逻辑

### 2. 资源要素解构层 (Resource Element Decomposition Layer)
- **有形资源**: 物理资产、设备、基础设施等
- **无形资源**: 数据、算法、品牌、知识等
- **人力资源**: 技能、经验、关系等
- **组织资源**: 流程、系统、文化等

### 3. 活动网络解构层 (Activity Network Decomposition Layer)
- **核心活动**: 直接创造价值的活动
- **支持活动**: 支撑核心活动的活动
- **管理活动**: 协调、监控、优化的活动
- **连接活动**: 协同、沟通、交互的活动

### 4. 连接关系解构层 (Connection Relationship Decomposition Layer)
- **物理连接**: 实物、资金、人员的流动
- **信息连接**: 数据、知识、信息的流动
- **价值连接**: 交易、合作、利益的关联
- **认知连接**: 认知、信任、文化的关联

### 5. 生态环境解构层 (Ecosystem Environment Decomposition Layer)
- **技术环境**: 技术发展趋势和应用可能性
- **市场环境**: 需求变化和竞争格局
- **政策环境**: 法规政策和制度环境
- **社会环境**: 文化、价值观和社会趋势
