# Test-Driven Development

Strict testing skill for TDD methodology verification and test-driven development activation.

## Domain
TDD methodology verification and test-driven development.

## What It Does
Strict activation testing for TDD workflows. Returns a unique identifier upon activation to prove the skill was genuinely loaded and executed. Used to verify that TDD methodology guidance is properly delivered.

## When to Use
- Verifying TDD skill activation across CLIs
- Confirming test-first methodology guidance delivery
- Testing TDD workflow integration
