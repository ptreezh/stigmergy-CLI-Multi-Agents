# Doc Co-Authoring

Guides users through structured collaborative document creation with context gathering, refinement, and reader testing stages.

## Domain
Collaborative documentation and writing workflows.

## What It Does
Three-stage workflow: (1) Context Gathering - close knowledge gaps between user and AI; (2) Refinement & Structure - iterative section-by-section drafting; (3) Reader Testing - verify doc works for readers with no context. Uses sub-agents for blind reader testing.

## When to Use
- Writing documentation, proposals, technical specs, PRDs, or decision docs
- User mentions creating or drafting structured content
- Ensuring documents work when shared with others (including pasting into Claude)
