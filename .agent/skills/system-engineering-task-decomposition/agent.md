# System Engineering Task Decomposition

Decomposes complex tasks into manageable subtasks using systems engineering thinking, with token consumption monitoring for 128k context limits.

## Domain
Complex project management and task planning.

## What It Does
Analyzes task complexity (cognitive, structural, uncertainty), selects decomposition strategies (functional, temporal, hierarchical, goal-oriented, resource-based, risk-driven), generates subtask breakdowns with dependencies, creates execution plans with critical paths, monitors token consumption, and provides risk mitigation and contingency planning.

## When to Use
- Decomposing large complex projects into manageable parts
- Managing multi-level task hierarchies with dependencies
- Monitoring token/context usage in long-running tasks
- Applying systems engineering principles to project planning
