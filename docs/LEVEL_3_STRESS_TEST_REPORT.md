## 🔍 真实测试机制说明

### 测试1的真相："10个并发CLI"

**标题说的**: "10个并发CLI模拟"

**实际做了什么**:
```javascript
// 不是启动真实的CLI进程
// 而是创建10个异步函数在同一Node.js进程中
for (let i = 0; i < 10; i++) {
  workers.push(simulateWorker(`worker-${i}`, store, i));
}

async function simulateWorker(workerId, store, index) {
  // 模拟worker行为
  await store.registerCLI(workerId, {...});
  await store.addEntry({...});
  // ...
}
```

**真相**:
- ❌ **没有启动真实的CLI进程**（claude, qwen, iflow等）
- ❌ **没有执行 `stigmergy use claude` 或类似命令**
- ✅ **只是10个异步函数在同一Node.js进程中并发执行**
- ✅ **它们都访问同一个SharedMemoryStore实例**

**验证了什么**:
- ✅ 文件锁在进程内并发下的安全性
- ❌ **未验证多进程并发场景**
- ❌ **未验证真实CLI进程协作**

**正确的验证等级**: Level 1 - 进程内并发测试

### 测试2的真相："100个并发任务"

**标题说的**: "100个并发任务处理"

**实际做了什么**:
```javascript
// 创建100个任务
for (let i = 0; i < 100; i++) {
  await store.enqueueTask({...});
}

// 然后启动3个worker处理
const workers = [];
for (let i = 0; i < 3; i++) {
  workers.push(startWorker(`worker-${i}`, store));
}
```

**真相**:
- ❌ **不是3个独立的CLI进程**
- ✅ **还是同一个Node.js进程内的3个异步函数**
- ✅ **但任务队列的并发安全性确实验证了**

**验证了什么**:
- ✅ 任务队列的原子性
- ✅ getNextTask()不会重复分配
- ❌ **未验证多CLI进程的任务分配**

**正确的验证等级**: Level 1.5 - 任务队列并发安全性

---

### 测试3的真相："大数据量性能"

**这个是真实的** ✅:
```javascript
for (let batch = 0; batch < 20; batch++) {
  for (let i = 0; i < 50; i++) {
    await store.addEntry({
      content: `...`,
      metadata: {...}
    });
  }
}
```

**真相**:
- ✅ 确实循环执行了写入操作
- ✅ 每次都触发文件锁、序列化、磁盘I/O
- ✅ 最终文件确实有1101条记录，192KB

**验证了什么**:
- ✅ 文件大小增长对性能的影响
- ✅ 大数据量下的写入性能
- ✅ 这是唯一真实的Level 2测试

**正确的验证等级**: Level 2 - 大数据量性能测试
