# 经验层级 3：详细说明
### 问题分析
在Node.js中，当有多个独立的异步操作时，如果使用await串行执行，
会导致总耗时 = 各个任务耗时之和。例如：3个各需1秒的任务串行执行需要3秒。

### 实施步骤
1. 识别哪些任务是独立的（无依赖关系）
2. 将这些任务包装为Promise
3. 使用 Promise.all([task1, task2, task3]) 并行执行
4. 使用 await 等待所有任务完成

### 代码示例
\`\`\`javascript
// 串行执行（慢）
const result1 = await asyncTask1(); // 1秒
const result2 = await asyncTask2(); // 1秒
const result3 = await asyncTask3(); // 1秒
// 总耗时: 3秒

// 并行执行（快）
const [result1, result2, result3] = await Promise.all([
  asyncTask1(),
  asyncTask2(),
  asyncTask3()
]);
// 总耗时: 1秒
\`\`\`

### 验证方法
- 对比执行时间
- 确认任务独立性
- 验证结果正确性

---

# 经验层级 4：上下文和变体
### 相关经验
- async/await模式
- Promise.race() 用于超时控制

### 变体和扩展
- Promise.allSettled() 处理部分失败
- Promise.race() 实现超时机制

### 注意事项
- 任务必须真正独立，不能有依赖关系
- 错误处理需要特别注意
- 并行执行会增加内存占用

## 🤖 自动技能生成

当经验积累到一定程度后，系统会自动生成新技能：

### 触发条件
- 同一主题的经验 >= 5条
- 经验的置信度 >= 70%

### 生成流程
1. 分析主题相关的所有经验
2. 使用LLM生成技能文档
3. 格式化为 agentskills.io 规范
4. 部署到 `~/.stigmergy/skills/`

### 生成的技能包含
- 技能元数据（YAML frontmatter）
- 用途说明
- 能力描述
- 使用方法
- 核心知识
- 最佳实践
- 常见模式

## 🔧 故障排除

### 问题1: CLI不可用
```
[LLM提取] CLI调用失败，使用基础分析
```
**解决**: 这是正常的，系统会降级到基础分析

### 问题2: 没有提取到经验
```
ℹ️ 未发现新的独立运行经验
```
**可能原因**:
- 最近没有CLI会话活动
- 会话内容不适合提取经验
- 会话文件格式不支持

**解决**: 先运行一些CLI会话，然后再提取

### 问题3: 技能生成失败
```
❌ 技能生成失败: 内容太短
```
**可能原因**:
- LLM调用失败
- 经验质量不够高
- 经验数量不足

**解决**: 确保有足够的CLI会话和高质量经验
