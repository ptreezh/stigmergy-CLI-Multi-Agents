## ⚠️ 之前营销材料的问题

### 问题 1: 定位模糊

**原文档问题**:
```
"Coordinate 8+ AI CLI tools"
```

**问题**:
- 用户会问："我为什么要协调多个 AI？"
- 没有回答"为什么需要 Stigmergy"
- 听起来像是为复杂而复杂

---

### 问题 2: 功能罗列而非价值传达

**原文档问题**:
```
Features:
- Smart Routing
- Cross-CLI Memory
- Remote Control
- Unified Skills
```

**问题**:
- 用户不关心功能，关心"对我有什么好处"
- 没有量化价值（省多少钱、节省多少时间）
- 没有使用场景对应

---

### 问题 3: 忽视竞争现实

**原文档问题**:
- 没有提及 OpenClaw 等 AI CLI
- 暗示 Stigmergy 是"独立工具"
- 用户发现需要额外安装 AI CLI 时会有被欺骗感

---

### 问题 4: 过度承诺

**原文档问题**:
```
"Boost productivity 300%+"
```

**问题**:
- 无数据支持
- 不同场景差异巨大（0% - 300%）
- 损害可信度

## ✅ 修正后的价值主张

### 核心价值主张（诚实版）

**Stigmergy 是什么**:
```
Stigmergy = AI CLI 编排层（依赖 OpenClaw 等工具）
```

**Stigmergy 的价值**:
```
1. 减少重复上下文 → 节省 Token 30-50%
2. 智能路由 → 选择最合适的 AI（成本/能力平衡）
3. 跨会话记忆 → 长期项目效率提升 80%
4. 远程编排 → 从聊天应用控制 AI
```

**Stigmergy 不是**:
```
❌ 不是 AI 工具（不生产 AI 能力）
❌ 不是 OpenClaw 替代品（是互补层）
❌ 不是万能药（简单任务无优势）
```
