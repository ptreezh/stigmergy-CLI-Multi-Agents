## 一、项目背景

### 根本性校正（2026-04-08）

**之前错误理解**:
- ❌ 考核操作效率：商品上架32分钟→4分钟
- ❌ 记录操作耗时、错误率
- ❌ CLI命令手册优先级最高

**正确理解（AI时代）**:
- ✅ 考核专业能力：商业理解、数据分析、决策判断
- ✅ AI帮操作，学生学专业
- ✅ 前后测对比：专业能力雷达图+提升报告

### 真实应用场景

| 角色 | 身份 | 核心需求 |
|------|------|----------|
| **教师** | 中职/高职电商专业教师 | AI辅助备课→一键生成实训案例→自动批改考核 |
| **学员** | 电商专业学生 | IM对话即可实训→真实商业决策训练→能力认证 |

## 二、核心能力验证

### 6维度专业能力模型

```
         商业理解力
            /\
           /  \
    数据分析  决策判断
          |    |
    问题诊断  创新思维
           \  /
            \/
         沟通表达力
```

| 维度 | 说明 | 考核方式 |
|------|------|----------|
| 商业理解力 | 理解电商运营逻辑 | 案例分析题 |
| 数据分析力 | 从数据中发现问题、提出建议 | 数据解读题 |
| 决策判断力 | 基于数据做合理决策 | 场景决策题 |
| 问题诊断力 | 快速定位问题根因 | 故障排查题 |
| 创新思维力 | 提出新的运营思路 | 开放创新题 |
| 沟通表达力 | 清晰汇报运营方案 | 方案汇报题 |

### 国家职业技能标准对标

| 综合能力分 | 国家等级 | 说明 |
|-----------|----------|------|
| 80+ | 三级/高级工 | 运营策划能力 |
| 60-79 | 四级/中级工 | 独立运营能力 |
| 40-59 | 五级/初级工 | 基础执行能力 |
| <40 | 未达标 | 需要加强学习 |

## 三、已完成模块

### 模块1: assessment-framework (评估框架)

| 文件 | 内容 |
|------|------|
| `SKILL.md` | 6维度能力模型+评分标准+示例题 |
| `assessment_framework.js` | 核心算法：评估/对比/雷达图/班级报告 |
| `test_framework.js` | 3个测试场景通过 |

**核心功能**:
- ✅ 6维度评估（0-100分/维度）
- ✅ 前后测对比（提升率计算）
- ✅ 雷达图生成
- ✅ 班级能力分布报告
- ✅ 国家等级对标

### 模块2: pre-assessment (前测)

| 文件 | 内容 |
|------|------|
| `SKILL.md` | 前测技能说明+30题设计 |
| `question_bank.js` | 30道题（商业理解10+数据分析10+决策判断10） |
| `test_pre_assessment.js` | 5个测试场景通过 |

**题库特点**:
- 选择题40% + 案例题40% + 开放题20%
- 每道题有评分标准（关键词匹配）
- 生成个人专业能力雷达图

### 模块3: post-assessment (后测)

| 文件 | 内容 |
|------|------|
| `SKILL.md` | 后测技能说明+对比报告 |
| `post_question_bank.js` | 30道题（同维度不同场景） |
| `test_post_assessment.js` | 4个测试场景通过 |

**核心差异**:
- 与前测题目完全不同（PQ1-PQ30 vs Q1-Q30）
- 同维度考核，避免记忆效应
- 生成前后测对比报告

### 模块4: course-runner (课程运行器)

| 文件 | 内容 |
|------|------|
| `SKILL.md` | 课程运行器技能说明 |
| `course_runner.js` | 核心模块：前测→实训→后测→报告 |
| `test_course_runner.js` | 5个测试场景通过 |

**核心流程**:
```
教师创建课程 → 学生前测(30题) → 实训(决策场景) → 学生后测(30题) → 生成报告
```

## 六、核心原则（来自顶级实训导师）

1. **AI帮操作，学生学专业** - 不考核操作速度，考核专业理解深度
2. **能力评估靠案例不靠操作** - 通过决策场景评估专业能力
3. **一堂好课的标准是"理解深"** - 学生能解释"为什么这么做"
4. **设计决策场景让学生思考** - 不是"怎么做"而是"为什么"
5. **能力认证必须有前后对比** - 专业能力雷达图对比
