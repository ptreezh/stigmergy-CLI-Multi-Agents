# WeChat iLink 10秒二维码有效期解决方案

> **问题**: iLink WeChat API的二维码有效期只有约10秒，导致用户扫码困难
> **解决**: 智能自动刷新机制 + 持续状态监控

---

## 🎯 问题分析

### 原始问题
```
用户反馈: "我已经扫码了，但微信对话没响应"
观察结果: 二维码在10-30秒内就过期，而不是预期的8分钟
```

### 根本原因
经过深入分析，发现：

1. **短轮询导致快速过期**: 原始监控脚本使用每2秒一次的短轮询
2. **API设计**: iLink API的二维码确实只有约10秒有效期
3. **状态检查失败**: 二维码过期后，状态查询失败

### 关键发现
```javascript
// ❌ 错误实现：短轮询
while (attempt < 240) {
  const status = await checkStatus(qrcode);
  await sleep(2000); // 频繁请求导致问题
}

// ✅ 正确实现：主动刷新
while (!scanned) {
  if (now - qrCode.timestamp >= 8000) { // 8秒后刷新
    qrCode = await getNewQrCode();
  }
  const status = await checkStatus(qrCode);
}
```

## 💡 解决方案

### 核心策略

**智能自动刷新 + 扫码状态追踪**

1. **定期刷新**: 每8秒自动获取新二维码
2. **状态追踪**: 一旦检测到扫码，停止刷新
3. **持续监控**: 扫码后持续监控确认状态
4. **清晰反馈**: 提供实时状态显示和操作指导

### 实现细节

#### 1. 二维码管理
```javascript
let currentQrCode = null;
let scanned = false; // 扫码状态标志
let qrCodeIndex = 0;

// 获取新二维码
async function getBotQrCode() {
  const response = await https.get(API_BASE + '/get_bot_qrcode?bot_type=3');
  return {
    qrcode: response.qrcode,
    qrcodeUrl: response.qrcode_img_content,
    timestamp: Date.now() // 记录生成时间
  };
}
```

#### 2. 智能刷新逻辑
```javascript
// 主动刷新策略
if (!scanned && now - currentQrCode.timestamp >= 8000) {
  console.log('🔄 二维码已过期，获取新二维码...');
  currentQrCode = await getBotQrCode();
  qrCodeIndex++;
  displayQrCode(currentQrCode, qrCodeIndex);
}
```

#### 3. 状态处理
```javascript
switch (statusResponse.status) {
  case 'wait':
    // 未扫码，继续监控
    process.stdout.write('.');
    break;

  case 'scaned':
    if (!scanned) {
      scanned = true; // 标记已扫码
      console.log('👀 已扫码！请在微信中确认登录');
    }
    break;

  case 'confirmed':
    // 登录成功
    return handleLoginSuccess(statusResponse);

  case 'expired':
    if (!scanned) {
      // 没扫码，刷新二维码
      currentQrCode = await getBotQrCode();
    } else {
      // 已扫码但过期，失败
      return null;
    }
    break;
}
```
