## 🔬 核心算法

### 1. Agent相似度计算

**算法**: 皮尔逊相关系数 (Pearson Correlation)

```javascript
相似度 = (Correlation + 1) / 2

其中 Correlation = Covariance(X,Y) / (σX × σY)
```

**优势**:
- 考虑评分的相对模式而非绝对值
- 对评分尺度差异不敏感
- 适合发现品味相似的Agents

**示例**:
```
Agent A: [5, 4, 3, 4, 5]
Agent B: [4, 5, 3, 4, 4]
相似度: 0.85 (高度相似)
```

### 2. 协同过滤推荐

**基于用户的协同过滤** (User-Based CF):

```
1. 找到与目标Agent相似的其他Agents
2. 收集这些相似Agents评价高的skills
3. 过滤掉目标Agent已使用的skills
4. 计算推荐分数 = 评分 × 相似度权重
5. 排序并推荐
```

**推荐分数计算**:
```javascript
WeightedScore = Σ(Rating_i × Similarity_i) / Σ(Similarity_i)
```

### 3. 跨领域推荐

**基于偏好模式的跨领域迁移**:

```
1. 分析Agent在源领域的偏好
   - 平均评分
   - 偏好的复杂度
   - 偏好的效果类型
   - Skill类型偏好

2. 在目标领域寻找匹配偏好的skills
   - 匹配度 = Σ(各维度匹配) / 维度数

3. 推荐高匹配度的skills
```

### 4. 安全核验机制

**多层安全检查**:

```javascript
安全核验流程:
1. 历史反馈检查
   - 是否有安全问题报告
   - 可靠性评分是否足够

2. 完整安全审计 (如果可用)
   - 恶意代码模式扫描
   - 依赖漏洞检测
   - 安全评分计算

3. 安全评分阈值
   - 必须达到60分以上
   - 否则不推荐
```

## 🎯 使用指南

### 基本使用

```javascript
// 初始化推荐系统
const recommender = new AutonomousSkillRecommender();

// 为Agent推荐skills
const report = await recommender.recommend('agent-123', {
  domain: 'security',
  useCase: 'audit',
  maxResults: 5
});

// 查看推荐结果
console.log(report.summary);
console.log(report.recommendations);
```

### 记录使用反馈

```javascript
// Agent使用skill后记录反馈
await recommender.recordUsage('soul-security-auditor', 'agent-123', {

  effectiveness: 'highly_effective',
  easeOfUse: 4,
  reliability: 5,
  performance: 5,

  suggestions: '可以增加更多检查规则',

});
```
