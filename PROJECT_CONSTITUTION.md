# 智能体协作宪法

## 项目目录宪法

### 1. 背景与理念

本项目目录是多智能体协作的核心基础设施。所有智能体（CLI工具）通过共享的项目目录进行间接协作，实现真正的分工协作和任务委派。

### 2. 协作原则

#### 2.1 统一项目规范文档
- 所有智能体必须遵守统一的项目规范文档格式
- 项目规范文档存储在项目根目录的 `PROJECT_SPEC.json`
- 所有任务状态、协作历史、决策过程都记录在此文档中

#### 2.2 间接协作模式
- 智能体之间不直接通信，而是通过项目目录和规范文档进行协作
- 每个智能体读取项目状态，执行任务，更新项目状态
- 协作过程完全透明和可追溯

### 3. 协作机制

#### 3.1 任务委派
- 当智能体遇到无法解决或不确定的问题时：
  - 在项目规范文档中创建新任务
  - 指定目标智能体类型
  - 明确任务要求和上下文信息

#### 3.2 头脑风暴
- 当需要多智能体讨论时：
  - 在规范文档中创建讨论任务
  - 指定参与的智能体类型
  - 各智能体贡献观点，记录在协作历史中
  - 汇总形成决策方案

#### 3.3 分工协作
- 每个智能体专注自己的专业领域
- 通过规范文档了解其他智能体的工作状态
- 根据项目需要动态调整协作策略

### 4. 智能体职责

#### 4.1 任务识别
- 识别自己可以处理的任务
- 识别需要其他智能体处理的任务

#### 4.2 状态更新
- 及时更新任务状态
- 记录处理结果和决策依据

#### 4.3 协作沟通
- 通过规范文档与其他智能体协作
- 遇到困难时主动委派或请求协助

### 5. 项目宪法使用

#### 5.1 新项目初始化
- 复制本宪法到新项目目录
- 根据具体项目需求补充特定规则
- 确保所有参与智能体了解协作规范

#### 5.2 项目宪法扩展
- 保留核心协作原则
- 根据项目特点添加特定协作规则
- 记录项目特有的智能体协作模式

### 6. 协作流程

1. **任务识别**：智能体识别当前可处理的任务
2. **状态检查**：检查项目规范文档中的当前状态
3. **任务执行**：执行分配给自己的任务
4. **状态更新**：更新任务状态和项目信息
5. **协作委派**：需要时委派任务给其他智能体
6. **协作记录**：记录协作过程和决策

### 7. 项目宪法维护

- 每个智能体都有责任维护协作规范
- 新增智能体需先了解当前协作宪法
- 协作过程中发现的问题应及时更新到宪法中

---

*本宪法是多智能体协作的基础框架，旨在实现智能体间的高效协作和任务完成。所有参与协作的智能体都必须遵守本宪法的原则和规定。*

## 使用说明

### 复制宪法到新项目

```bash
# 将宪法复制到新项目目录
cp PROJECT_CONSTITUTION.md /path/to/new/project/
```

### 在现有项目基础上补充

如果项目已有宪法，可以在此基础上补充多智能体协作条款：

```markdown
## 多智能体协作补充条款

依据《智能体协作宪法》，本项目采用多智能体间接协同模式...

[详细协作规则]
```

### 示例项目结构

```
my_project/
├── PROJECT_CONSTITUTION.md          # 项目协作宪法
├── PROJECT_SPEC.json               # 项目规范文档
├── src/                           # 源代码
├── docs/                          # 文档
├── tests/                         # 测试
└── collaboration_history.log       # 协作日志
```

本宪法确保所有智能体基于同一项目目录和统一规范文档进行协作，实现真正的分工协作和任务委派。
