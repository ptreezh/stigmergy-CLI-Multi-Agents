## 三、架构问题与改进建议

**现状**：
- `coordination-layer.config.json` 存在但功能不清晰
- 跨 CLI 通信依赖简单的 shell 命令 (`stigmergy distcli`)
- 缺少真正的协调层实现

**改进建议**：

```typescript
// 建议：实现真正的协调层服务
class CoordinationLayer {
  // 1. 分布式消息总线
  messageBus: EventEmitter;
  
  // 2. 共享状态管理
  sharedState: Map<string, any>;
  
  // 3. 任务调度器
  taskScheduler: TaskScheduler;
  
  // 4. CLI 注册与发现

}
```

**具体行动**：
1. 实现基于 Redis/SQLite 的共享状态存储
2. 添加任务队列和优先级调度
3. 支持 CLI 自动发现和注册

---

### 🔴 问题 2: 技能系统碎片化

**现状**：
- 技能分散在多个位置：
  - `~/.qwen/skills/` (50+ 技能)
  - `~/.qwen/skills/skills/` (30+ 技能)
  - `~/.stigmergy/skills/` (外部)
  - `C:\stigmergy-CLI-Multi-Agents\skills` (外部)
- 技能格式不统一（有的用 `SKILL.md`，有的用 `skill.md`）
- 缺少技能版本管理和依赖关系

**改进建议**：

```yaml

---
name: writing-plans
version: 2.1.0
description: 创建详细实施计划
author: stigmergy
dependencies:
  - brainstorming@^1.0.0
  - planning-with-files@^1.0.0
triggers:
  - "创建计划"
  - "write plan"
  - "planning"
capabilities:
  - multi-step-planning
  - file-awareness
  - checkpoint-support
---
```

**具体行动**：
1. 统一技能目录结构
2. 实现技能包管理器 (`stigmergy skill install/publish`)
3. 添加技能依赖解析
