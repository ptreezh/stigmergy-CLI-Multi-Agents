### 二、架构问题诊断

#### 🔴 **严重问题**

1. **技能加载机制脆弱**
   - `skills-hook.js` 依赖 `scripts/` 子目录结构，但大多数技能只有 `SKILL.md`
   - 无 JavaScript 实现时返回 `{ type: 'directory' }`，无法实际执行
   - 错误处理仅打印日志，无 fallback 机制

2. **Cross-CLI 通信未真正启用**
   - `config.json` 配置了 8 种 CLI，但实际只实现了 `qwen_hook.js`
   - `distcli` 命令依赖 stigmergy CLI，但路径硬编码
   - 无实际跨进程通信实现

3. **安全执行器过度限制**
   - `qwen_safe_executor.js` 阻止 `stigmergy.*superpowers:` 模式
   - 但 `superpowers-integration.md` 又要求通过 stigmergy 调用技能
   - 自相矛盾的设计

4. **Soul 系统空心化**
   - `soul-reflection.md` 等是伪代码文档，非可执行技能
   - 依赖 `~/.stigmergy/history/` 目录，但无实际数据收集机制
   - 进化状态文件路径分散，无统一管理

#### 🟡 **中等问题**

5. **技能发现路径混乱**
   ```json
   // config.json 中多处配置不一致
   "skill_discovery_paths": [
     "C:\\Users\\WIN10\\.stigmergy\\skills",
     "C:\\Users\\WIN10\\.qwen\\skills",
     "C:\\stigmergy-CLI-Multi-Agents\\skills"
   ]
   ```

6. **Hook 优先级无实际作用**
   - `hooks.json` 定义了 priority，但无调度器使用

7. **Session-Start 注入路径错误**
   - `SKILLS_ROOT = path.join(os.homedir(), 'skills')` 
   - 实际技能在 `~/.qwen/skills/` 或 `~/.stigmergy/skills/`
