## Stigmergy 性能分析与优化建议

### 当前架构状态

根据配置文件和文档分析，Stigmergy v1.2.1 是一个多 Agent CLI 系统，具有以下特点：

1. **技能发现机制**：5 个技能搜索路径，支持自动刷新
2. **跨 CLI 通信**：支持 8 种 CLI 工具协作
3. **Hook 系统**：支持会话钩子和自动化
4. **Superpowers 集成**：技能别名和自动发现

### 性能瓶颈识别

#### 1. 技能加载性能
- **问题**：`skill_discovery_paths` 配置了 5 个路径，每次刷新需扫描所有目录
- `skill_cache_ttl: 300000` (5 分钟) 可能导致频繁刷新
- **优化建议**：
  - 实现增量扫描（仅监控变化的文件）
  - 增加缓存 TTL 或采用惰性加载
  - 使用文件系统监听替代轮询

#### 2. 跨 CLI 通信开销
- **问题**：`communication_timeout: 30000ms` 较长，串行调用时累积延迟
- **优化建议**：
  - 并行调用独立 CLI（参考 `dispatching-parallel-agents` 技能）
  - 实现请求去重和结果缓存
  - 超时时间动态调整

#### 3. Hook 系统开销
- **问题**：Hook 触发时同步执行，阻塞主流程
- **优化建议**：
  - 异步执行非关键 Hook
  - Hook 执行结果缓存
  - 限制 Hook 执行时间

#### 4. 技能执行模式
- **问题**：技能调用前需检查 1% 可能性规则，导致过度调用
- **优化建议**：
  - 实现技能调用热度统计
  - 智能预加载高频技能
  - 技能依赖图优化

### 具体优化方案

#### 阶段 1：缓存优化
```
1. 技能列表缓存（TTL 延长至 30 分钟）
2. 技能内容哈希缓存（仅重新加载变化文件）
3. CLI 调用结果缓存（相同请求直接返回）
```

#### 阶段 2：并行化改进
```
1. 技能扫描并行化（多路径同时扫描）
2. 跨 CLI 调用并行 dispatch
3. Hook 异步执行队列
```

#### 阶段 3：资源管理
```
1. 技能实例池（避免重复初始化）
2. CLI 连接复用（减少启动开销）
3. 内存缓存上限控制
```

### 监控建议

建议添加性能监控指标：
- 技能加载时间
- CLI 调用响应时间
- Hook 执行耗时
- 缓存命中率

这些优化可以显著提升 Stigmergy 的响应速度和资源利用效率。
