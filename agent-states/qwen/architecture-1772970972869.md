### 三、架构问题与改进建议 🔧

#### 问题 1: **版本碎片化**
```
config.json:        "stigmergy_version": "1.2.1"
session-start.ts:   STIGMERGY_VERSION = "1.3.76"
```
**影响**: 版本不一致可能导致功能兼容性问题

**建议**:
- 建立单一版本源 (single source of truth)
- 使用语义化版本管理 (semver)
- 添加版本迁移脚本

---

#### 问题 2: **技能发现路径冗余**
```json
"skill_discovery_paths": [
  "C:\\Users\\WIN10\\.stigmergy\\skills",
  "C:\\Users\\WIN10\\.qwen\\skills",
  "C:\\stigmergy-CLI-Multi-Agents\\skills",
  "C:\\Users\\WIN10\\.stigmergy\\skills\\skills",  // 重复
  "C:\\Users\\WIN10\\.qwen\\skills\\skills"       // 重复
]
```

**建议**:
- 移除重复路径
- 实现技能去重机制 (基于技能 name/hash)
- 添加技能优先级配置

---

#### 问题 3: **双 Agent 循环缺少可视化监控**
当前所有进化机制都依赖双 Agent 循环，但缺少:
- 执行状态追踪
- 性能指标收集
- 失败恢复机制

**建议**: 添加 `evolution-monitor` 技能
```markdown
~/.stigmergy/monitor/
├── dashboard.md       # 实时状态面板
├── metrics.jsonl      # 性能指标日志
└── alerts/            # 异常告警
```

---

#### 问题 4: **知识融合机制不完善**
Co-Evolve 的知识融合步骤描述较简单:
```
→ 去重 → 优先级排序 → 识别互补点
```

**建议**: 实现更精细的知识融合算法
```python
def merge_knowledge(sources):
    # 1. 语义相似度检测 (embedding-based)
    # 2. 冲突检测 (contradiction detection)
    # 3. 来源可信度加权
    # 4. 时间衰减因子
    # 5. 生成融合知识图谱
```

---

#### 问题 5: **竞争进化的评估维度单一**
当前评估主要依赖主观评分:
```
→ 正确性评分 → 效率评分 → 质量评分 → 可维护性评分
```

**建议**: 添加客观指标
| 维度 | 指标示例 |
|------|----------|
| 正确性 | 测试通过率、边界覆盖度 |
| 效率 | 执行时间、内存占用 |
| 质量 | 代码复杂度、重复率 |
| 可维护性 | 文档完整度、修改影响范围 |

---

#### 问题 6: **存储结构缺少版本控制**
```
~/.stigmergy/soul-state/
├── evolutions/
├── evolved-skills/
└── evolution-log.jsonl
```

**建议**:
- 使用 SQLite/LevelDB 替代文件存储
- 支持进化历史回滚
- 添加知识快照功能

- 状态同步机制
- 冲突解决策略
