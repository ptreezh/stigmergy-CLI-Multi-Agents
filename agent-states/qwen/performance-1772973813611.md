## Stigmergy 性能分析与优化建议

### 一、当前架构分析

#### 1. 核心组件

| 组件 | 功能 | 性能影响 |
|------|------|----------|
| `hooks/*.js` | CLI 钩子系统 | 每次会话启动时加载 |
| `skills/` | 技能系统 | 动态加载，缓存机制 |
| `auto-memory-hook.js` | 自动记忆 | 实时监控 stdout，会话结束处理 |
| `session-start.ts` | 会话初始化 | 注入上下文 |
| `qwen_safe_executor.js` | 安全执行器 | 命令过滤和超时控制 |

#### 2. 识别的性能瓶颈

**🔴 高优先级问题：**

1. **技能加载效率低** (`skills-hook.js`)
   - 每次初始化都遍历整个技能目录
   - 没有实现缓存机制
   - 同步文件操作阻塞主线程

2. **内存泄漏风险** (`auto-memory-hook.js`)
   - `sessionMessages` 数组持续增长无限制
   - 钩住 stdout 可能影响性能
   - 会话结束处理可能超时

3. **重复代码** 
   - `qwen_safe_executor.js` 和 `qwen_hook.js` 有重叠逻辑
   - 多个 hook 文件重复加载相同配置

4. **技能发现路径冗余** (`config.json`)
   - 5 个技能搜索路径，部分可能重复
   - 每次会话可能重复扫描

**🟡 中优先级问题：**

5. **会话启动注入过多上下文** (`session-start.ts`)
   - 读取多个文件注入上下文
   - 可能增加初始响应延迟

6. **安全过滤正则效率** (`qwen_safe_executor.js`)
   - 多个正则表达式顺序执行
   - 可优化为单次扫描

// auto-memory-hook.js 优化
class AutoMemoryHook {
  constructor() {
    this.MAX_MESSAGES = 1000; // 限制消息数量
    this.MAX_MEMORY_MB = 50;  // 限制内存使用
  }

  hookStdout() {
    const self = this;
    process.stdout.write = function(chunk) {
      const text = chunk.toString();
      
      if (text.trim()) {
        self.sessionMessages.push({
          timestamp: Date.now(),
          content: text
        });

        // 内存保护：移除旧消息
        if (self.sessionMessages.length > self.MAX_MESSAGES) {
          self.sessionMessages = self.sessionMessages.slice(-self.MAX_MESSAGES);
        }
      }
      
      return originalWrite.apply(process.stdout, arguments);
    };
  }
}

// lib/command-safety.js
class CommandSafety {
  constructor() {
    this.blockedPatterns = [
      /superpowers:.*[^a-zA-Z0-9_-]/i,
      /stigmergy.*superpowers:.*/i,
      /Bash\(["'])stigmergy.*superpayments:.*/i,
    ];
    this.safeSuperpowers = new Set([ // 使用 Set 提高查找效率
      'using-git-worktrees',
      'writing-plans',
      // ... 其他安全技能
    ]);
  }

  isSafe(command) {
    return !this.blockedPatterns.some(p => p.test(command)) &&
           this.checkSuperpower(command);
  }

  checkSuperpower(command) {
    const match = command.match(/superpowers:([a-zA-Z0-9_-]+)/i);
    if (!match) return true;
    return this.safeSuperpowers.has(match[1].toLowerCase());
  }
}

module.exports = new CommandSafety();
