# Stigmergy 架构分析报告

## 一、架构概述

**Stigmergy** 是一个基于**痕迹刺激**（Stigmergy）理论的多 Agent 协作 CLI 系统，版本 1.2.1。核心理念是通过共享的痕迹（文件、记忆、技能）实现多 CLI 间的间接协作。

### 核心组件

| 组件 | 位置 | 功能 |
|------|------|------|
| **Hooks** | `~/.qwen/hooks/` | 会话拦截、技能注入、安全过滤 |
| **Skills** | `~/.qwen/skills/` | 可复用能力模块（Markdown 定义） |
| **Extensions** | `~/.qwen/extensions/` | 自动记忆、Superpowers 集成 |
| **Config** | `~/.qwen/config.json` | 跨 CLI 配置、安全策略 |

---

## 二、架构优势 ✅

1. **双 Agent 循环机制** - 主 Agent + Subagent 的派发/审查模式确保质量
2. **技能驱动设计** - 通过 Markdown 技能文件实现能力模块化
3. **自动记忆系统** - 会话结束自动提取经验并存储
4. **多 CLI 协作** - 支持 Claude、Gemini、Qwen 等 8 种 CLI
5. **安全过滤层** - `qwen_safe_executor.js` 拦截危险命令

### 问题 3: 双 Agent 循环缺少超时/重试机制

**现状**: `two-agent-loop` 无错误恢复

```
派发 Subagent → Code Review → 修复 → 循环
// 问题：无限循环风险、无超时控制
```

**改进方案**:

```javascript
class TwoAgentLoop {
  async executeWithRetry(task, maxRetries = 3, timeout = 300000) {
    let attempts = 0;
    
    while (attempts < maxRetries) {
      try {
        const result = await Promise.race([
          this.dispatchSubagent(task),
          this.timeout(timeout)
        ]);
        
        const review = await this.codeReview(result);
        if (review.passed) return result;
        
        task = this.createFixTask(result, review.issues);
        attempts++;
      } catch (error) {
        if (error.code === 'TIMEOUT') {
          console.warn(`⚠️ 任务超时，尝试 ${attempts + 1}/${maxRetries}`);
        }
      }
    }
    
    throw new Error(`任务失败：超过最大重试次数`);
  }
}
```

```javascript
// 动态超时 - 基于任务复杂度
function calculateTimeout(task) {
  const baseTimeout = 30000;
  
  const complexityFactors = {
    'code-generation': 2.0,
    'code-review': 1.5,
    'testing': 2.5,
    'documentation': 1.0
  };
  
  const factor = complexityFactors[task.type] || 1.0;
  return baseTimeout * factor * (task.linesOfCode / 100 || 1);
}

// 添加熔断器
class CircuitBreaker {
  constructor(threshold = 5, timeout = 60000) {
    this.failures = 0;
    this.threshold = threshold;
    this.state = 'CLOSED';
  }
  
  async execute(fn) {
    if (this.state === 'OPEN') {
      throw new Error('熔断器开启，拒绝执行');
    }
    try {
      return await fn();
    } catch (e) {
      this.failures++;
      if (this.failures >= this.threshold) {
        this.state = 'OPEN';
        setTimeout(() => this.state = 'HALF_OPEN', this.timeout);
      }
      throw e;
    }
  }
}
```

```javascript
// 添加指标收集
class MetricsCollector {
  constructor() {
    this.metrics = {
      skillInvocations: new Counter('skill_invocations_total'),
      subagentLatency: new Histogram('subagent_duration_seconds'),
      errorRate: new Gauge('error_rate'),
      memoryUsage: new Gauge('memory_usage_bytes')
    };
  }
  
  recordSkillInvocation(skillName, duration, success) {
    this.metrics.skillInvocations.inc({ skill: skillName });
    this.metrics.subagentLatency.observe(duration);
    if (!success) {
      this.metrics.errorRate.inc({ skill: skillName });
    }
  }
}

app.get('/metrics', (req, res) => {
  res.send(metricsCollector.export());
```
