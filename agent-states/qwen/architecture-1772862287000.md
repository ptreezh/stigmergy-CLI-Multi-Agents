### 三、核心机制分析

#### 1. 双 Agent 循环 (Two-Agent Loop) ⭐

**这是所有进化的基础执行机制**

```
主 Agent (当前 CLI)
   │
   ├──→ 1. 派发 Subagent (执行具体任务)
   │         └──→ 执行任务 → 返回结果
   │
   ├──→ 2. Code Review (质量审查)
   │         └──→ 反馈 → 改进
   │
   ├──→ 3. 迭代修复 (如有问题)
   │
   └──→ 4. 循环直到完成
```

**关键原则**：
- 每个子任务后必须审查
- 问题立即修复（不过夜）
- 不并行派发（避免冲突）
- 使用 TDD

#### 2. 四种进化模式

| 模式 | 依赖 | 流程 | 触发 |
|------|------|------|------|
| **soul-reflection** | two-agent-loop | 数据收集→深度分析→改进建议 | 关键词/定时/手动 |
| **soul-evolution** | two-agent-loop | 模式分析→知识提取→技能创建→验证测试 | 关键词/周定时 |
| **soul-co-evolve** | two-agent-loop | 知识发布→知识获取→知识融合→能力提升 | 关键词/天定时 |
| **soul-compete** | two-agent-loop | 方案生成→独立执行→评估对比→选择进化 | 关键词/决策时 |

#### 3. Superpowers 集成

通过 `stigmergy skill` 命令调用高级功能：

```python
class KnowledgeConflictDetector:
    def __init__(self):
        self.conflict_types = {
            "contradiction": self.detect_contradiction,
            "redundancy": self.detect_redundancy,
            "obsolescence": self.detect_obsolescence
        }
    
    def detect_conflicts(self, new_knowledge, existing_knowledge_base):
        """检测新知识与现有知识库的冲突"""
        conflicts = []
        for conflict_type, detector in self.conflict_types.items():
            result = detector(new_knowledge, existing_knowledge_base)
            if result:
                conflicts.append({
                    "type": conflict_type,
                    "severity": result["severity"],
                    "details": result["details"],
                    "resolution_suggestion": result["resolution"]
                })
        return conflicts
    
    def resolve_conflict(self, conflict, resolution_strategy="merge"):
        """解决知识冲突"""
        if resolution_strategy == "merge":
            return self.merge_knowledge(conflict)
        elif resolution_strategy == "version":
            return self.create_knowledge_version(conflict)
        else:
            return self.manual_resolution_required(conflict)
```
