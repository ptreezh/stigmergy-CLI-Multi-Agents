## Stigmergy 架构分析报告

### 一、架构概述

**Stigmergy** 是一个多 Agent 协作 CLI 系统，基于**痕迹协调**(Stigmergy) 理论实现 AI Agent 间的间接协作。核心思想是通过共享环境和文件痕迹实现 Agent 间的协调，而非直接通信。

### 二、当前架构层次

### 三、核心机制分析

#### 1. 双 Agent 循环 (Two-Agent Loop)
```
主 Agent → 派发 Subagent → 执行任务 → Code Review → 迭代修复 → 循环完成
```
**问题**：当前实现依赖手动触发，缺少自动化调度机制。

#### 2. 进化系统 (Soul Evolution)
- **自主进化**：基于反思创建新技能
- **协同进化**：多 CLI 知识共享
- **竞争进化**：多方案对比选择

**问题**：存储路径混乱 (`~/.stigmergy/soul-state/` vs 项目本地)

#### 3. 技能发现机制
```json
{
  "skill_discovery_paths": [
    "C:\\Users\\WIN10\\.stigmergy\\skills",
    "C:\\Users\\WIN10\\.qwen\\skills",
    "C:\\stigmergy-CLI-Multi-Agents\\skills"
  ]
}
```
**问题**：硬编码路径，跨平台兼容性差。
