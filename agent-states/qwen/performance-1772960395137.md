#### 1. 技能加载优化 (高优先级)

**问题**：`skills-hook.js` 每次会话都重新扫描整个技能目录

```javascript

async loadSkills() {
  const skillDirs = fs.readdirSync(this.skillsDir);
  for (const skillName of skillDirs) {
    // 每次都重新读取...
  }

```

**优化方案**：添加 TTL 缓存 + 增量更新

```javascript
// 优化后 - 带缓存和增量更新
class QwenSkillsHook {
  constructor() {
    this.skillCache = new Map();
    this.cacheTimestamp = 0;
    this.CACHE_TTL = 5 * 60 * 1000; // 5 分钟
    this.skillHashes = new Map(); // 文件哈希缓存
  }

  async loadSkills() {
    const now = Date.now();
    
    // 缓存未过期，跳过
    if (now - this.cacheTimestamp < this.CACHE_TTL) {
      return;
    }

    // 检查技能目录是否变化
    const dirHash = await this.getDirectoryHash(this.skillsDir);
    if (dirHash === this.lastDirHash) {
      this.cacheTimestamp = now;
      return;
    }

    // 仅重新加载变化的技能
    await this.incrementalLoadSkills();
    this.cacheTimestamp = now;
    this.lastDirHash = dirHash;
  }
}
```

**预期收益**：会话启动时间减少 **60-80%**

#### 2. 并行 Code Review (高优先级)

**问题**：`two-agent-loop` 强制串行执行，Subagent 完成后必须等待 Review

```
当前流程:
Subagent 执行 → Review → 修复 → Review → 下一任务
     5 分钟      2 分钟   3 分钟   2 分钟    5 分钟
总计：17 分钟/任务
```

**优化方案**：允许独立任务并行 Review

```
优化后流程:
任务 A: Subagent 执行 → Review ─┐
任务 B: Subagent 执行 → Review   ├─→ 并行执行
任务 C: Subagent 执行 → Review ─┘
总计：max(任务 A, 任务 B, 任务 C) ≈ 7-10 分钟
```

**实现建议**：
- 在 `subagent-driven-development` 中添加 `parallelReview: true` 选项
- 对于不共享状态的 Review，允许并行派发

**预期收益**：多任务场景时间减少 **40-60%**
