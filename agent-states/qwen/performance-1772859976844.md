### 二、识别的性能问题

**问题：**
- 技能发现路径重复配置（`config.json` 中 5 个搜索路径）
- 技能缓存 TTL 仅 300000ms (5 分钟)，导致频繁重新扫描
- 没有按需加载机制，所有技能在启动时扫描

**优化建议：**

#### 2. **子代理调用开销大**

**问题：**
- `subagent-driven-development` 每个任务启动 3 个子代理（实现 + 规范审查 + 代码质量审查）
- 每个子代理独立启动，无上下文复用
- 审查循环导致多次迭代

**优化建议：**
```markdown
# 当前流程
任务 → 实现子代理 → 规范审查 → (修复循环) → 代码质量审查 → (修复循环) → 完成

# 优化方案
1. 上下文预加载：控制器预先读取所有相关文件，一次性传递给子代理
2. 并行审查：规范审查和代码质量审查可并行进行
3. 审查合并：单次审查同时检查规范符合性和代码质量
4. 增量审查：仅审查变更文件，而非整个代码库

# 预期收益
- 减少 50% 子代理调用次数
- 减少 70% 文件 I/O 开销
- 缩短 60% 整体执行时间
```

#### 3. **安全过滤器性能开销**

**问题：**
- `qwen_safe_executor.js` 中每个命令进行 5 个正则匹配
- 实时输出过滤增加延迟
- 正则表达式未预编译

**优化建议：**
```javascript
// 当前实现 - 每次执行时创建正则
this.blockedCommands = [
  /superpowers:.*[^a-zA-Z0-9_-]/i,
  /stigmergy.*superpowers:.*/i,
  // ...
];

// 优化实现 - 预编译正则
this.blockedCommands = [
  new RegExp('superpowers:.*[^a-zA-Z0-9_-]', 'i'),
  new RegExp('stigmergy.*superpowers:.*', 'i'),
  // ...
].map(r => r.compile());  // 预编译

// 使用单一复合正则替代多个独立正则
this.blockedPattern = new RegExp(
  '(superpowers:[^a-zA-Z0-9_-])|' +
  '(stigmergy.*superpowers:.*)|' +
  '(Bash\\(["\']stigmergy.*superpowers:.*)',
  'i'
);
```

#### 4. **配置同步机制缺失**

**问题：**
- `config.json`、`hooks.json`、`settings.json` 独立维护
- `qwen_config_patcher.js` 每次运行重写配置
- 无配置变更检测机制

**优化建议：**
```javascript
// 添加配置版本控制
config: {
  version: "1.2.1",
  config_schema_version: 2,
  last_patch_time: "2025-12-15T07:03:38.210Z",
  patch_history: [
    { patch: "superpowers_support", applied: "2025-12-15T07:03:38.210Z" }
  ]
}

// 增量更新而非全量重写
async patchConfigFile() {
  const config = await this.readConfig();
  const patchesNeeded = this.detectMissingPatches(config);
  
  if (patchesNeeded.length === 0) {
    console.log('✅ 配置已是最新');
    return;
  }
  
  for (const patch of patchesNeeded) {
    await this.applyPatch(config, patch);
  }
}
```
