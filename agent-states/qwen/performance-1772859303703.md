### 识别的性能问题

#### 1. **Skills 发现机制效率低**
```json
"skill_discovery_paths": [
  "C:\\Users\\WIN10\\.stigmergy\\skills",
  "C:\\Users\\WIN10\\.qwen\\skills",
  "C:\\stigmergy-CLI-Multi-Agents\\skills",
  "C:\\Users\\WIN10\\.stigmergy\\skills\\skills",  // 冗余路径
  "C:\\Users\\WIN10\\.qwen\\skills\\skills"        // 冗余路径
]
```
**问题**：5 个搜索路径，存在冗余嵌套路径，每次技能发现需遍历所有目录

**优化建议**：
- 移除冗余路径（`skills/skills`）
- 实现技能缓存机制（当前 `skill_cache_ttl: 300000` 即 5 分钟合理）
- 添加技能索引文件避免全量扫描

#### 2. **ESM/CommonJS 模块兼容性错误**
从日志中看到的致命错误：
```
Error [ERR_REQUIRE_ESM]: require() of ES Module ... inquirer ... not supported
```
**问题**：`src/core/installer.js` 使用 CommonJS `require()` 加载 ESM 模块 `inquirer`

**优化建议**：
```javascript
// 当前 (错误)
const inquirer = require('inquirer');

// 修复方案 A: 动态 import()
const inquirer = await import('inquirer');

// 修复方案 B: 降级到 CommonJS 版本
const inquirer = require('inquirer@8');
```

#### 3. **Hooks 安全过滤器性能**
`qwen_safe_executor.js` 使用多个正则表达式：
```javascript
/stigmergy.*superpowers:.*/i,
/Bash\(["'])stigmergy.*superpowers:.*/i,
/node.*stigmergy.*superpowers:.*/i,
```
**问题**：每个命令需匹配多个正则，复杂模式影响性能

**优化建议**：
- 合并正则表达式为单一模式
- 使用字符串前缀检查替代正则（快 10-100 倍）
```javascript
// 优化后
if (cmd.includes('stigmergy') && cmd.includes('superpowers:')) {
  // block
}
```

#### 4. **Cross-CLI 超时配置**
```json
"timeout": 30,
"communication_timeout": 30000
```
**问题**：30 秒超时对复杂任务可能不足，导致重试开销

**优化建议**：
- 实现动态超时（基于任务类型）
- 添加超时指数退避重试机制
