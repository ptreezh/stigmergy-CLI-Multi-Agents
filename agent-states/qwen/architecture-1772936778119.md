#### 2. **Hook 系统耦合度高**

**问题**：
- `session-start.ts` 混合版本检查、技能注入、状态读取
- `auto-memory-hook.js` 直接 hook `stdout.write`，易冲突
- Hook 之间无依赖管理

**改进建议**：
```typescript
// 建议：Hook 管道模式
interface Hook {
  name: string;
  priority: number;  // 执行顺序
  dependencies?: string[];  // 依赖的 Hook
  initialize(ctx: Context): Promise<void>;
  onMessage(msg: Message): Promise<Message>;
  onExit(ctx: Context): Promise<void>;
}

class HookPipeline {
  private hooks: Hook[] = [];

  register(hook: Hook) {
    // 拓扑排序处理依赖
    this.hooks.push(hook);
    this.hooks.sort((a, b) => a.priority - b.priority);
  }

  async execute(phase: 'initialize' | 'onMessage' | 'onExit', ctx: any) {
    for (const hook of this.hooks) {
      await hook[phase](ctx);
    }
  }
}
```

#### 3. **安全模型过于简单**

**问题**：
- `qwen_safe_executor.js` 使用正则匹配，易绕过
- `blocked_patterns` 可被编码/拼接绕过

**改进建议**：
```javascript
// 建议：AST 级命令解析 + 沙箱
class SecureExecutor {
  constructor() {
    this.policyEngine = new PolicyEngine();
    this.auditLogger = new AuditLogger();
  }

  async execute(command) {
    // 1. AST 解析命令结构
    const ast = this.parseCommand(command);

    // 2. 策略引擎验证
    const decision = await this.policyEngine.evaluate({
      command: ast,
      context: this.getContext(),
      riskLevel: this.assessRisk(ast)
    });

    // 3. 审计日志
    this.auditLogger.log({
      timestamp: Date.now(),
      command,
      decision,
      context: this.getContext()
    });

    // 4. 沙箱执行
    if (decision.allowed) {
      return await this.sandboxExecute(ast, decision.constraints);
    }
    throw new SecurityError(`Blocked: ${decision.reason}`);
  }
}
```

**问题**：
- `two-agent-loop` 只是文档，无实际执行框架
- Subagent 派发依赖 CLI 原生 `task` 工具
- 无标准化的 Review 协议

**改进建议**：
```javascript
// 建议：实现完整的双 Agent 循环框架
class TwoAgentLoop {
  async execute(task) {
    const subtasks = this.decompose(task);
    const results = [];

    for (const subtask of subtasks) {
      // 1. 派发 Subagent
      const subagentResult = await this.dispatch({
        description: subtask.description,
        prompt: this.buildPrompt(subtask),
        constraints: subtask.constraints
      });

      // 2. Code Review
      const review = await this.review({
        artifact: subagentResult,
        criteria: this.getReviewCriteria(subtask.type)
      });

      // 3. 迭代修复
      if (!review.passed) {
        const fixedResult = await this.fix({
          original: subagentResult,
          feedback: review.feedback,
          maxIterations: 3
        });
        results.push(fixedResult);
      } else {
        results.push(subagentResult);
      }
    }

    return this.synthesize(results);
  }
}
```

```typescript
// 建议：基于 IPC/RPC 的通信层
class CrossCliCommunication {
  private connections: Map<string, RpcConnection> = new Map();

  async connect(cliName: string) {
    if (!this.connections.has(cliName)) {
      const conn = await this.createRpcConnection(cliName);
      this.connections.set(cliName, conn);
    }
    return this.connections.get(cliName)!;
  }

  async invoke(cliName: string, method: string, params: any) {
    const conn = await this.connect(cliName);
    return await conn.invoke(method, params, {
      timeout: this.getTimeout(cliName),
      retry: 3
    });
  }

  async broadcast(method: string, params: any) {
    return await Promise.allSettled(
      this.supportedClis.map(cli => this.invoke(cli, method, params))
    );
  }
}
```
