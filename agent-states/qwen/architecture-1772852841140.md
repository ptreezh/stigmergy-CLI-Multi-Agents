## 📊 Stigmergy 架构分析

### 当前架构概览

**Stigmergy** 是一个 CLI 多智能体协作系统，核心设计理念是通过**痕迹刺激 (Stigmergy)** 机制实现多个 AI CLI 工具之间的间接协作。

### 核心组件

| 组件 | 位置 | 功能 |
|------|------|------|
| **配置系统** | `config.json` | 跨 CLI 支持、安全设置、Superpowers 集成 |
| **钩子系统** | `hooks/` | Qwen 集成、安全执行器、配置补丁 |
| **技能系统** | `hooks/skills/` | soul-reflection、soul-auto-evolve |
| **技能发现** | 多路径搜索 | `~/.stigmergy/skills`, `~/.qwen/skills` 等 |

## 🔍 识别的问题

### 1. **架构混乱与重复配置**
```
问题：配置分散在多个文件中
- config.json (主配置)
- hooks.json (钩子配置)
- skill-access-config.json (技能访问配置)
```
**影响**: 配置不一致风险高，维护困难

### 2. **安全机制过于激进**
```javascript
// qwen_safe_executor.js 中的问题
blockedCommands: [
  /superpowers:.*[^a-zA-Z0-9_-]/i,  // 过于宽泛

]
```
**影响**: 合法功能被误杀，用户体验差
