## 📊 Stigmergy 架构分析报告

### 一、当前架构概览

**Stigmergy** 是一个多 Agent CLI 协作系统，核心设计理念是通过**痕迹刺激**（Stigmergy）机制实现多个 AI CLI 之间的间接协作。

#### 核心组件

| 组件 | 位置 | 功能 |
|------|------|------|
| **Cross-CLI Adapter** | `hooks.json` | 支持 8 种 CLI 工具间的通信 |
| **Skill System** | `~/.stigmergy/skills/` | 可扩展的技能系统 |
| **Hook System** | `hooks/` | 会话钩子、安全执行器 |
| **Reflection System** | `~/.stigmergy/reflection/` | 自我反思与进化 |
| **Superpowers** | 通过 `stigmergy skill` 调用 | 高级功能快捷方式 |

### 二、已识别的架构问题

#### 🔴 严重问题

1. **ESM/CommonJS 模块冲突**
   ```
   Error [ERR_REQUIRE_ESM]: require() of ES Module ... inquirer ... not supported
   ```
   - `src/core/installer.js` 使用 CommonJS `require()` 导入 ESM 模块 `inquirer`

2. **技能部署不完整**
   - `superpowers` 等基础技能在安装后未正确部署到 `~/.stigmergy/skills/`
   - 配置中的技能路径混乱（5 个不同路径）

3. **安全机制过度限制**
   - `qwen_safe_executor.js` 中 blocked patterns 过于宽泛
   - 可能阻止合法的 stigmergy 命令

#### 🟡 中等问题

4. **配置冗余与不一致**
   - `config.json` 和 `hooks.json` 中存在重复配置
   - `cross_cli_adapter` 和 `cross_cli_communication` 功能重叠

5. **技能发现机制低效**
   - 硬编码的 5 个技能搜索路径
   - 无缓存失效策略（TTL 5 分钟可能不足）

6. **Session Hook 路径错误**
   ```javascript
   const SKILLS_ROOT = path.join(os.homedir(), 'skills');
   // 应该是 ~/.stigmergy/skills 或 ~/.qwen/skills
   ```

#### 🟢 轻微问题

7. **文档分散**
   - `CROSS_CLI_GUIDE.md` 过于简单
   - 缺少架构文档和 API 参考

8. **触发模式有限**
   ```javascript
   "trigger_patterns": [
     "use\\s+(\\w+)\\s+to\\s+(.+)$",
     "call\\s+(\\w+)\\s+(.+)$",
     // 缺少更多自然语言触发方式
   ]
   ```
