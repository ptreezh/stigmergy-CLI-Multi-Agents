## 📊 Stigmergy 性能分析与优化建议

### 一、当前架构性能瓶颈分析

#### 1. **技能加载与发现机制**
**问题识别：**
- 技能发现路径过多（5 个路径），每次启动都扫描
- `skill_cache_ttl: 300000` (5 分钟) 可能导致技能更新延迟
- 技能文件采用 Markdown 格式，每次使用需解析

**优化建议：**
```
1. 实现技能索引预构建
   - 启动时生成技能索引缓存
   - 仅当技能文件变化时重建索引
   
2. 懒加载技能内容
   - 仅加载技能元数据（name, description, tags）
   - 实际使用时再加载完整内容
   
3. 增量刷新机制
   - 监听技能目录文件变化
   - 仅刷新变化的技能
```

#### 2. **双 Agent 循环性能开销**
**问题识别：**
- 每个子任务都进行 Code Review，增加 2-3 倍调用次数
- 顺序执行，无法并行化独立任务
- 审查循环可能导致多次迭代

**优化建议：**
```
1. 分级审查策略
   - 简单任务：仅自审查
   - 中等任务：快速审查（单轮）
   - 复杂任务：完整双轮审查

2. 并行派发优化
   - 识别真正独立的子任务
   - 使用 dispatching-parallel-agents 并行执行
   - 合并审查结果

3. 审查缓存
   - 相似代码模式复用审查结果
   - 建立审查规则库
```

#### 3. **Subagent 上下文管理**
**问题识别：**
- 每个 subagent 需要完整上下文传递
- 计划文件需重复读取
- 任务间上下文不共享，导致重复分析

**优化建议：**
```
1. 上下文池化
   - 建立共享上下文存储
   - Subagent 按需读取上下文片段
   - 避免重复传递相同信息

2. 任务上下文预取
   - 控制器预先提取所有任务上下文
   - 建立上下文索引
   - Subagent 快速定位所需信息

3. 增量上下文更新
   - 仅传递变化的上下文
   - 使用 diff 方式更新
```

#### 4. **技能触发机制**
**问题识别：**
- 技能触发基于关键词匹配，可能误触发
- 1% 规则导致过度使用技能
- 技能调用前需检查所有可能技能

**优化建议：**
```
1. 智能技能推荐
   - 基于任务类型自动推荐
   - 机器学习技能使用模式
   - 减少不必要的技能检查

2. 技能优先级队列
   - 建立技能优先级评分
   - 高优先级技能先检查
   - 低概率技能延迟检查

3. 技能组合优化
   - 识别常用技能组合
   - 批量加载相关技能
   - 减少技能切换开销
```
