## Stigmergy 架构分析

### 当前架构概述

**Stigmergy** 是一个多 AI Agent 协作 CLI 系统，核心设计理念是通过"痕迹刺激"（Stigmergy）机制实现多个 AI CLI 工具之间的协作。

#### 核心组件

| 组件 | 位置 | 功能 |
|------|------|------|
| **Cross-CLI 适配器** | `hooks.json` | 支持 8 种 CLI 工具间的通信 |
| **技能系统** | `skills/` | 可扩展的技能库，支持 40+ 技能 |
| **安全执行器** | `qwen_safe_executor.js` | 阻止不安全命令执行 |
| **配置补丁** | `qwen_config_patcher.js` | 自动修补 CLI 配置 |
| **会话钩子** | `session-start.ts` | 注入 superpowers 上下文 |

#### 支持的 CLI 工具
```
claude, gemini, qwen, iflow, qodercli, codebuddy, copilot, codex
```

### 架构问题识别

#### 🔴 严重问题

1. **ESM/CommonJS 模块冲突**
   ```
   Error: require() of ES Module ... inquirer/dist/index.js 
   from ... stigmergy/src/core/installer.js not supported
   ```
   - `inquirer` v9+ 使用 ESM，但 `installer.js` 使用 CommonJS `require()`

2. **技能发现路径混乱**
   ```json
   "skill_discovery_paths": [
     "C:\\Users\\WIN10\\.stigmergy\\skills",
     "C:\\Users\\WIN10\\.qwen\\skills",
     "C:\\stigmergy-CLI-Multi-Agents\\skills",
     "C:\\Users\\WIN10\\.stigmergy\\skills\\skills",  // 重复嵌套
     "C:\\Users\\WIN10\\.qwen\\skills\\skills"        // 重复嵌套
   ]
   ```

3. **安全规则过于严格**
   - `qwen_safe_executor.js` 中的正则模式可能误杀合法命令
   - `blockedPatterns` 阻止所有 `superpowers:.*` 但同时又依赖 superpowers

#### 🟡 中等问题

4. **文档分散**
   - `CROSS_CLI_GUIDE.md` 过于简单
   - 缺少架构设计文档
   - 技能使用说明分散在各 `skill.md` 文件中

5. **技能版本管理缺失**
   - 技能有 `version` 字段但无版本控制机制
   - 无技能兼容性检查

6. **Hook 系统不完善**
   - `session-start.ts` 路径硬编码为 `~/skills` 而非实际路径
   - 缺少错误恢复机制

#### 🟢 轻微问题

7. **配置冗余**
   - `config.json` 中 `superpowers_support` 和 `skill_discovery` 配置重复
   - `hooks.json` 中 `trigger_patterns` 与 `supported_patterns` 功能重叠

8. **缺少测试框架**
   - 无自动化测试验证跨 CLI 通信
   - 技能加载无单元测试

### 改进建议

#### 优先级 1：修复关键 Bug

```markdown
## 1.1 修复 ESM/CommonJS 冲突
- 将 `src/core/installer.js` 改为 ESM 模块
- 或降级 `inquirer` 到 v8 (CommonJS 兼容版本)
- 添加 `package.json` 的 `"type": "module"` 声明

## 1.2 清理技能发现路径
- 移除重复的嵌套路径
- 添加路径验证逻辑，跳过不存在的路径
- 使用环境变量支持自定义路径

## 1.3 重构安全规则
- 使用白名单而非黑名单机制
- 添加安全规则配置文件
- 提供安全规则绕过机制（需用户确认）
```

#### 优先级 2：架构优化

```markdown
## 2.1 统一配置管理
- 合并 `superpowers_support` 和 `skill_discovery`
- 添加配置验证 Schema (使用 Zod 或 Joi)
- 支持配置热重载

## 2.2 改进 Hook 系统
- 使用相对路径而非硬编码
- 添加 Hook 执行超时控制
- 支持 Hook 链式调用

## 2.3 技能生命周期管理
- 添加技能加载/卸载 API
- 实现技能依赖解析
- 支持技能版本兼容性检查
```

#### 优先级 3：增强功能

```markdown
## 3.1 文档系统
- 创建 `ARCHITECTURE.md` 描述整体设计
- 添加 `CONTRIBUTING.md` 贡献指南
- 生成技能 API 参考文档

## 3.2 测试框架
- 添加 Jest/Vitest 测试配置
- 创建跨 CLI 通信模拟测试
- 实现技能加载集成测试

## 3.3 监控与日志
- 添加结构化日志 (使用 Pino/Winston)
- 实现性能指标收集
- 创建错误追踪机制
```

#### 优先级 4：用户体验

```markdown
## 4.1 CLI 交互优化
- 添加交互式命令补全
- 实现命令历史搜索
- 提供命令执行预览

## 4.2 错误处理改进
- 友好的错误消息（非堆栈跟踪）
- 错误恢复建议
- 添加 `--verbose` 调试模式

## 4.3 性能优化
- 技能缓存机制
- 懒加载技能
- 并行加载独立技能
```
