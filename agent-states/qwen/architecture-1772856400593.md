### 二、核心组件分析

#### 1. Cross-CLI 通信机制
- **实现方式**: 通过 `stigmergy distcli "Prompt"` 命令
- **支持 CLI**: claude, gemini, qwen, iflow, qodercli, codebuddy, copilot, codex
- **配置位置**: `~/.qwen/config.json` 中的 `cross_cli_communication`

#### 2. Skills 系统
- **存储位置**: 多路径发现 (`~/.stigmergy/skills`, `~/.qwen/skills`, etc.)
- **技能格式**: Markdown 文件，包含 YAML frontmatter
- **调用方式**: `stigmergy skill call <skill-name> "<prompt>"`

#### 3. Hooks 系统
- **会话钩子**: `session-start.ts` 用于注入 superpowers
- **安全钩子**: `qwen_safe_executor.js` 过滤危险命令
- **配置钩子**: `qwen_config_patcher.js` 自动修补配置

#### 4. Agents 系统
- **位置**: `~/.qwen/agents/agents/`
- **类型**: 领域专家（ANT 专家、场域分析专家、扎根理论专家等）
- **触发**: 基于任务类型自动选择

### 三、架构问题识别

#### 🔴 严重问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **1. 循环依赖风险** | Qwen 配置中禁止 `stigmergy.*superpowers`，但 superpowers 又依赖 stigmergy | 功能冲突 |
| **2. 安全机制过度限制** | `qwen_safe_executor.js` 阻止所有 superpowers 调用 | 核心功能失效 |
