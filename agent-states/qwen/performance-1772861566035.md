## Stigmergy 性能分析与优化建议

### 📊 当前架构性能特征

#### 1. **双 Agent 循环机制性能分析**

**当前模式：**
```
主 Agent → Subagent 执行 → Code Review → 修复 (如有问题) → 循环
```

**性能瓶颈：**
- **串行执行**：每个子任务必须等待前一个完成 + 审查完成
- **审查开销**：每个任务至少 2 次 subagent 调用（执行 + 审查），最多 4 次（执行 + 审查 + 修复 + 再审）
- **上下文切换**：每次 subagent 派发都有上下文加载成本

**优化建议：**

| 优化点 | 当前 | 建议 | 预期提升 |
|--------|------|------|----------|
| 审查触发 | 每次必审 | 基于风险分级审查 | 减少 30-50% 审查调用 |
| 简单任务 | 完整双循环 | 单次 subagent 执行 | 减少 50% 调用 |
| 审查阈值 | 无 | 设定变更行数/复杂度阈值 | 减少不必要审查 |
