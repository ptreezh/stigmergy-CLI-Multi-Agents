### 当前架构分析

根据配置文件和代码结构，Stigmergy CLI 是一个多智能体协作系统，主要特点：

1. **技能系统**: 基于 Markdown 的技能定义 (`*.md`)

### 潜在性能瓶颈

#### 1. **技能发现与加载**
```json
// 当前配置
"skill_discovery_paths": [
  "C:\\Users\\WIN10\\.stigmergy\\skills",
  "C:\\Users\\WIN10\\.qwen\\skills",
  "C:\\stigmergy-CLI-Multi-Agents\\skills",
  "C:\\Users\\WIN10\\.stigmergy\\skills\\skills",
  "C:\\Users\\WIN10\\.qwen\\skills\\skills"
],
"skill_cache_ttl": 300000,  // 5 分钟
"refresh_on_change": true
```

**问题**:
- 5 个搜索路径，每次技能发现需要遍历多个目录
- `refresh_on_change: true` 可能导致频繁的文件系统监控
- 缓存 TTL 5 分钟可能导致技能更新后延迟

#### 2. **跨 CLI 通信超时**
```json
"communication_timeout": 30000,  // 30 秒
"timeout": 30
```

**问题**:
- 30 秒超时对于简单任务可能过长
- 没有分级超时机制（简单查询 vs 复杂任务）

#### 3. **安全过滤性能**
从 `qwen_safe_executor.js` 可见：
```javascript
/stigmergy.*superpowers:.*/i,
/Bash\(["'])stigmergy.*superpowers:.*/i,
```
- 多个正则表达式在每次命令执行时都需要匹配
