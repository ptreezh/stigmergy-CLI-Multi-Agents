# Stigmergy 架构分析报告

## 一、架构概览

Stigmergy 是一个**多 AI CLI 协作系统**，通过 stigmergy（协作标记）机制实现多个 AI 助手之间的协同工作。

### 核心组件

| 组件 | 位置 | 功能 |
|------|------|------|
| **配置系统** | `config.json` | 跨 CLI 配置、技能发现、安全设置 |
| **Hooks 系统** | `hooks/` | 会话启动、技能注入、安全执行 |
| **Skills 系统** | `skills/` | 50+ 技能模块，包括 Superpowers、Soul 进化系列 |
| **Agents 系统** | `agents/agents/` | 24 个专业智能体（ANT、场域分析、扎根理论等） |
| **Extensions** | `extensions/` | 自动记忆、Superpowers 集成 |
| **安全系统** | `qwen_safe_executor.js` | 命令过滤、Superpowers 访问控制 |

## 二、架构优势

### ✅ 已实现的优势

1. **跨 CLI 协作机制**

## 三、架构问题诊断

### 🔴 关键问题

#### 1. **模块耦合度过高**
```
问题：session-start.ts 同时处理版本检查、技能注入、上下文管理
影响：单点故障风险，难以测试和维护
```

#### 2. **技能加载机制不统一**
```javascript
// skills-hook.js 中混合了多种加载逻辑
- 从 scripts/ 子目录加载
- 从根目录加载 JS 文件
- 回退到目录注册
问题：缺乏统一的技能接口规范
```

#### 3. **安全系统碎片化**
```
- qwen_safe_executor.js：Qwen 专用
- config.json 中的安全设置
- hooks 中的模式匹配
问题：安全策略分散，难以审计和更新
```

#### 4. **状态管理缺失**
```
- 会话状态分散在内存、文件系统
- 缺乏统一的状态持久化机制
- 跨会话恢复依赖 resumesession 技能
```

#### 5. **错误处理不一致**
```javascript
// 多处使用静默失败
try { ... } catch (e) { /* 静默 */ }
问题：难以诊断问题，错误信息丢失
```

### 方案 3：集中式安全策略

```javascript
// security/policy-manager.js
class SecurityPolicyManager {
  constructor() {
    this.policies = new Map();
    this.auditLog = new AuditLog();
  }
  
  registerPolicy(name, policy) {
    this.policies.set(name, policy);
  }
  
  async validate(command) {
    const results = [];
    for (const [name, policy] of this.policies) {
      const result = await policy.validate(command);
      results.push({ name, ...result });
    }
    return this.aggregateResults(results);
  }
}
```

### 方案 4：状态管理

```javascript
// state/state-store.js
class StateStore {
  constructor() {
    this.memory = new Map();
    this.persistence = new FilePersistence();
  }
  
  async set(key, value, options = {}) {
    this.memory.set(key, value);
    if (options.persist) {
      await this.persistence.save(key, value);
    }
  }
  
  async get(key, options = {}) {
    if (this.memory.has(key)) {
      return this.memory.get(key);
    }
    if (options.loadFromPersistence) {
      return await this.persistence.load(key);
    }
    return null;
  }
}
```

## 七、总结

Stigmergy 架构具有**创新的跨 CLI 协作理念**和**丰富的技能生态系统**，但在**模块化**、**统一接口**和**状态管理**方面存在改进空间。建议按优先级逐步实施上述改进方案，以提升系统的可维护性、安全性和扩展性。
